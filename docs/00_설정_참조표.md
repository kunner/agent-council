# 설정 참조표 (단일 진실 공급원)

> 이 문서는 프로젝트 전체에서 사용되는 모든 설정값의 **단일 진실 공급원**입니다.
> 다른 문서에서는 이 파일을 참조하고 값을 중복 정의하지 마십시오.

---

## 1. 서버 포트

| 서비스 | 포트 |
|---|---|
| Council Server | `3001` |
| Uptime Kuma | `3002` |
| Vite Dev Server | `5173` |
| Grafana Alloy | `12345` |

---

## 2. 서버 배포 경로

| 용도 | 경로 |
|---|---|
| 배포 루트 | `/opt/agent-council/` |
| 작업 공간 | `/opt/agent-council/workspace/{projectId}/` |
| 로그 | `/opt/agent-council/logs/` |

---

## 3. Firestore 컬렉션 경로

모든 프로젝트 데이터는 `projects` 하위에 위치합니다.

```
users/{userId}
projects/{projectId}
projects/{projectId}/git/config
projects/{projectId}/rooms/{roomId}
projects/{projectId}/rooms/{roomId}/messages/{messageId}
projects/{projectId}/sets/{setId}
projects/{projectId}/sets/{setId}/logs/{logId}
projects/{projectId}/tasks/{taskId}
projects/{projectId}/pullRequests/{prId}
projects/{projectId}/snapshots/{snapshotId}
usage/{userId}/monthly/{YYYY-MM}
```

---

## 4. Git Worktree 구조 (bare repo 방식)

```
/opt/agent-council/workspace/{projectId}/
├── .git-repo/      # bare repository
├── main/           # main worktree
└── set-{id}/       # set worktree (세트별 생성)
```

---

## 5. tmux 세션 명명

```
council-{projectId}-{setId}
```

## 7. Firebase 설정

| 항목 | 값 |
|---|---|
| Firestore 리전 | `asia-northeast3` (서울) |
| Auth 무료 한도 | 월 10,000 MAU (소셜 로그인) |
| Firestore Spark | 일 50K 읽기 / 20K 쓰기 / 1GB 저장 |
| Storage Spark | 5GB / 일 1GB 다운로드 |

---

## 8. Claude Code 세션

| 항목 | 값 |
|---|---|
| 세션당 예상 메모리 | ~1.5–2GB |
| 동시 세션 상한 | 8 (Oracle 24GB 기준) |
| CLI 어댑터 파일명 | `ClaudeCliAdapter.ts` |
| 채널 어댑터 파일명 | `ClaudeChannelAdapter.ts` |

---

## 9. 공유 Enum/타입 정규 정의

> 정의 위치: `packages/shared`

| Enum | 값 |
|---|---|
| `ProjectStatus` | `'planning' \| 'in_progress' \| 'review' \| 'completed' \| 'paused' \| 'archived'` |
| `SetStatus` | `'idle' \| 'working' \| 'waiting' \| 'done' \| 'error'` |
| `TaskStatus` | `'backlog' \| 'in_progress' \| 'review' \| 'done' \| 'blocked'` |
| `PRStatus` | `'open' \| 'reviewing' \| 'approved' \| 'merged' \| 'closed'` |
| `ReviewStatus` | `'requested' \| 'in_review' \| 'changes_requested' \| 'approved'` |
| `SessionStatus` | `'starting' \| 'active' \| 'idle' \| 'sleeping' \| 'stopping' \| 'stopped' \| 'error'` |
| `MessageSenderType` | `'human' \| 'leader' \| 'system'` |
| `SystemEventType` | 프로토콜 문서의 통합 이벤트 타입 전체 목록 참조 |
| `SnapshotTrigger` | `'pr_merged' \| 'task_done' \| 'manual' \| 'scheduled' \| 'session_end' \| 'pm_away'` |
| `RoomPurpose` | `'main' \| 'design' \| 'bug_triage' \| 'code_review' \| 'integration' \| 'custom'` |

---

## 10. UI 설정

| 항목 | 값 |
|---|---|
| 칸반 열 표시명 | `Backlog \| In Progress \| Review \| Done \| Blocked` |
| 위저드 최대 단계 | 5단계 (유형별 3~5) |
| Nginx 보안 헤더 | `X-Frame-Options DENY` |

---

## 11. 타임스탬프 형식

| 컨텍스트 | 형식 |
|---|---|
| Firestore 저장 | Firebase `Timestamp` 객체 |
| REST API 응답 | ISO 8601 string (예: `2025-01-01T00:00:00.000Z`) |
| WebSocket 페이로드 | Unix ms `number` (예: `1735689600000`) |
