# 메시지 프로토콜 설계

## 개요

Agent Council의 메시지 프로토콜은 세 가지 통신 경로를 다룬다.

1. **Council Server ↔ Claude Code** — 리더에게 컨텍스트를 주입하고 응답을 수신하는 AI 세션 통신
2. **Council Server ↔ Web Client (Firestore)** — 영속 메시지 저장 및 실시간 동기화
3. **Council Server ↔ Web Client (WebSocket)** — 휘발성 상태 이벤트 (타이핑 인디케이터, 프로그레스)

---

## 1. 메시지 타입 정의

Council Room의 모든 메시지는 `senderType` 필드로 구분된다.

```typescript
type SenderType = 'human' | 'leader' | 'system'
```

| 타입 | 발신자 | 설명 |
|---|---|---|
| `human` | PM (사람) | 웹 UI에서 입력하는 PM의 메시지 |
| `leader` | AI 리더 | Claude Code 세션에서 생성된 리더의 응답 |
| `system` | Council Server | Git 이벤트, PR 알림, 태스크 변경 등 자동 알림 |

---

## 2. 메시지 기본 스키마

모든 메시지가 공유하는 베이스 타입:

```typescript
interface BaseMessage {
  id: string                    // Firestore 자동 생성 문서 ID
  roomId: string                // 소속 Council Room ID
  projectId: string             // 소속 프로젝트 ID
  senderType: SenderType
  senderId: string              // human: Firebase UID | leader: setId | system: 'council-server'
  senderName: string            // 표시 이름 ("나", "백엔드팀", "시스템")
  content: string               // 메시지 본문 (Markdown 지원)
  replyTo?: string              // 답장 대상 메시지 ID
  isInternalLog: boolean        // Set 내부 로그 요약 메시지 여부 (접기/펼치기)
  timestamp: Timestamp          // Firestore Timestamp
  metadata?: MessageMetadata    // 타입별 확장 메타데이터
}
```

### 2.1 메시지 메타데이터 스키마

```typescript
interface MessageMetadata {
  // 아티팩트 (리더 메시지에서 주로 사용)
  artifacts?: ArtifactRef[]

  // 태스크 참조
  taskRefs?: string[]           // 관련 태스크 ID 목록

  // Git 관련
  commitHash?: string           // 관련 커밋 해시 (단일)
  commitHashes?: string[]       // 관련 커밋 해시 목록
  pullRequestUrl?: string       // 관련 PR URL
  pullRequestId?: string        // Firestore pullRequests 문서 ID
  branch?: string               // 관련 브랜치명

  // 토큰 사용량 (리더 메시지)
  tokenUsage?: TokenUsage

  // 인라인 액션 버튼
  actions?: InlineAction[]

  // 시스템 메시지 전용
  systemEvent?: SystemEventPayload

  // 세션 복원 관련
  sessionRestored?: boolean
  snapshotId?: string
}

interface ArtifactRef {
  path: string                  // 파일 경로 (worktree 기준 상대 경로)
  type: 'created' | 'modified' | 'deleted'
  language?: string             // 코드 하이라이팅용 언어 힌트
  storageUrl?: string           // Firebase Storage URL (선택적 미리보기)
}

interface TokenUsage {
  inputTokens: number
  outputTokens: number
  totalTokens: number
  cacheReadTokens?: number      // 프롬프트 캐시 히트
  cacheWriteTokens?: number
}

interface InlineAction {
  id: string
  label: string                 // 버튼 표시 텍스트 (예: "PR 보기")
  type: 'link' | 'command' | 'task_create' | 'review_request' | 'merge'
  payload: Record<string, unknown>
}
```

## 3. 메시지 타입별 상세

### 3.1 Human 메시지

PM이 웹 UI에서 입력하는 메시지. 구조가 가장 단순하다.

```typescript
interface HumanMessage extends BaseMessage {
  senderType: 'human'
  senderId: string              // Firebase Auth UID
  senderName: string            // Firebase displayName
  metadata?: {
    taskRefs?: string[]         // 태스크 언급 시
    actions?: InlineAction[]    // 현재 미사용, 확장 여지
  }
}
```

**JSON 예시:**
```json
{
  "id": "msg_001",
  "roomId": "room_main",
```

### 3.3 System 메시지

Council Server가 자동으로 생성하는 알림 메시지. `systemEvent` 서브타입으로 세분화된다.

```typescript
interface SystemMessage extends BaseMessage {
  senderType: 'system'
  senderId: 'council-server'
  senderName: '시스템'
  metadata: {
    systemEvent: SystemEventPayload
    actions?: InlineAction[]
    taskRefs?: string[]
    pullRequestId?: string
    pullRequestUrl?: string
    commitHashes?: string[]
    branch?: string
  }
}
```

#### 대화 히스토리 선별 로직

전체 메시지를 그대로 주입하면 컨텍스트 윈도우를 빠르게 소진한다. 다음 우선순위로 선별한다.

```typescript
interface HistorySelectionConfig {
  maxMessages: number           // 기본 30
  maxTokenEstimate: number      // 기본 8000 (시스템 프롬프트 공간 확보)
}

function selectConversationHistory(
  allMessages: BaseMessage[],
  config: HistorySelectionConfig = { maxMessages: 30, maxTokenEstimate: 8000 }
): ClaudeMessage[] {
  // 1순위: 시스템 메시지 중 중요 이벤트 (pr_created, pr_merged, task_updated)
  // 2순위: Human 메시지 전부 (PM 지시는 누락하면 안 됨)
  // 3순위: 다른 Leader 메시지 (최근 순)
  // 4순위: 자기 자신의 이전 메시지 (최근 순, 중복 응답 방지)
  // → 토큰 추정치 초과 시 오래된 leader/system 메시지부터 제거

  const prioritized = [
    ...allMessages.filter(m => m.senderType === 'human'),
    ...allMessages.filter(m =>
      m.senderType === 'system' &&
      ['pr_created', 'pr_merged', 'task_created', 'task_updated'].includes(
        m.metadata?.systemEvent?.type ?? ''
      )
    ),
    ...allMessages.filter(m => m.senderType === 'leader'),
  ]

  // 타임스탬프 기준 정렬 후 최근 N개 선택
  return prioritized
    .sort((a, b) => a.timestamp.toMillis() - b.timestamp.toMillis())
    .slice(-config.maxMessages)
    .map(m => ({
      role: m.senderType === 'human' ? 'user' : 'assistant',
      content: formatMessageForClaude(m),
    }))
}

function formatMessageForClaude(message: BaseMessage): string {
  const prefix = message.senderType === 'leader'
    ? `[${message.senderName}]`
    : message.senderType === 'system'
    ? '[시스템]'
    : '[PM]'
  return `${prefix} ${message.content}`
}
```

```typescript
function parseLeaderResponse(
  rawText: string,
  claudeUsage: ClaudeCliResponse['usage']
): Partial<LeaderMessage> {
  // JSON 메타 블록 추출 시도
  const metaMatch = rawText.match(/```json\s*(\{[\s\S]*?"__council_meta"[\s\S]*?\})\s*```/)
  let metadata: Partial<MessageMetadata> = {}
  let content = rawText

  if (metaMatch) {
    try {
      const meta: LeaderResponseMetadataBlock = JSON.parse(metaMatch[1])
      metadata = {
        artifacts: meta.artifacts,
        commitHashes: meta.commits,
        branch: meta.branch,
        taskRefs: meta.taskRefs,
        actions: meta.actions,
      }
      // 메타 블록 제거 후 본문만 남김
      content = rawText.replace(metaMatch[0], '').trim()
    } catch {
      // 파싱 실패 — 원문 그대로 사용
    }
  }

  if (claudeUsage) {
    metadata.tokenUsage = {
      inputTokens: claudeUsage.input_tokens,
      outputTokens: claudeUsage.output_tokens,
      totalTokens: claudeUsage.input_tokens + claudeUsage.output_tokens,
      cacheReadTokens: claudeUsage.cache_read_input_tokens,
      cacheWriteTokens: claudeUsage.cache_creation_input_tokens,
    }
  }

  return { content, metadata }
}
```

```typescript
// 클라이언트 연결 URL
// wss://council.yourdomain.com/ws?projectId={id}&roomId={id}&token={firebase_jwt}

// 클라이언트 측 재연결 정책
const WS_RECONNECT_DELAYS = [1000, 2000, 5000, 10000, 30000]  // ms, 지수 백오프

class CouncilWebSocket {
  private ws: WebSocket | null = null
  private reconnectAttempt = 0

  connect(url: string): void {
    this.ws = new WebSocket(url)

    this.ws.onmessage = (event) => {
      const wsEvent: WebSocketEvent = JSON.parse(event.data)
      this.handleEvent(wsEvent)
    }

    this.ws.onclose = () => {
      // WebSocket 연결 끊김 → Firestore 리스너는 계속 동작
      // 휘발성 이벤트만 누락되므로 UI 기능에 치명적이지 않음
      this.scheduleReconnect(url)
    }
  }

  private scheduleReconnect(url: string): void {
    const delay = WS_RECONNECT_DELAYS[
      Math.min(this.reconnectAttempt, WS_RECONNECT_DELAYS.length - 1)
    ]
    this.reconnectAttempt++
    setTimeout(() => this.connect(url), delay)
  }
}
```

```typescript
class LeaderResponseQueue {
  private queue: Map<string, Promise<void>> = new Map()  // roomId → 처리 중인 Promise
```

## 9. 대화 컨텍스트 구성 상세

리더에게 전달할 메시지를 선별하는 완전한 로직:

```typescript
interface ContextBuildConfig {
  setId: string                 // 컨텍스트를 받을 리더의 Set ID
  maxMessages: number           // 기본 30
  maxEstimatedTokens: number    // 기본 8000
  alwaysIncludeTypes: SystemEventType[]  // 항상 포함할 시스템 이벤트
}

const DEFAULT_CONFIG: ContextBuildConfig = {
  setId: '',
  maxMessages: 30,
  maxEstimatedTokens: 8000,
  alwaysIncludeTypes: ['pr_created', 'pr_merged', 'task_created', 'task_updated', 'rebase_conflict'],
}

interface ScoredMessage {
  message: BaseMessage
  score: number                 // 높을수록 우선 포함
  estimatedTokens: number
}

function buildConversationContext(
  allMessages: BaseMessage[],
  config: ContextBuildConfig
): ClaudeMessage[] {
  const scored: ScoredMessage[] = allMessages.map(msg => ({
    message: msg,
    score: scoreMessage(msg, config),
    estimatedTokens: estimateTokens(msg.content),
  }))

  // 점수 내림차순 정렬
  scored.sort((a, b) => b.score - a.score)

  // 토큰 예산 내에서 선택
  const selected: BaseMessage[] = []
  let tokenBudget = config.maxEstimatedTokens

  for (const { message, estimatedTokens } of scored) {
    if (selected.length >= config.maxMessages) break
    if (tokenBudget - estimatedTokens < 0) continue
    selected.push(message)
    tokenBudget -= estimatedTokens
  }

  // 시간 순 정렬 후 Claude 포맷으로 변환
  return selected
    .sort((a, b) => a.timestamp.toMillis() - b.timestamp.toMillis())
    .map(m => ({
      role: m.senderType === 'human' ? 'user' : 'assistant',
      content: formatMessageForClaude(m),
    }))
}

function scoreMessage(msg: BaseMessage, config: ContextBuildConfig): number {
  let score = 0

  // 기본 점수: 최신일수록 높음 (Unix timestamp 기반 정규화)
  const ageMinutes = (Date.now() - msg.timestamp.toMillis()) / 60000
  score += Math.max(0, 100 - ageMinutes * 0.5)

  // 타입별 가중치
  if (msg.senderType === 'human') score += 200           // PM 지시는 최우선
  if (msg.senderType === 'leader' && msg.senderId === config.setId) score += 50  // 자기 메시지

  // 중요 시스템 이벤트
  if (msg.senderType === 'system') {
    const eventType = msg.metadata?.systemEvent?.type
    if (config.alwaysIncludeTypes.includes(eventType as SystemEventType)) {
      score += 300  // 항상 포함
    }
  }

  // 태스크 참조 있으면 가중치 추가
  if ((msg.metadata?.taskRefs?.length ?? 0) > 0) score += 30

  return score
}

function estimateTokens(text: string): number {
  // 영문 기준 ~4자/토큰, 한국어 기준 ~2자/토큰 (혼합 텍스트 보수적 추정)
  return Math.ceil(text.length / 2.5)
}
```

```typescript
async function persistMessageBatch(messages: BaseMessage[]): Promise<void> {
```
