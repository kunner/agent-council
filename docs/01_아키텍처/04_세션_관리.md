# 세션 관리 & 대화 재개

## 개요

Claude Code 세션은 기본적으로 **휘발성**이다. 브라우저를 닫거나, 서버가 재시작되거나, 장시간 방치되면 세션 컨텍스트가 사라진다. Agent Council은 이를 해결하기 위해 3개 레이어로 대화 연속성을 보장한다.

```
Layer 1: 대화 히스토리        ← Firestore 자동 보존 (항상 복구 가능)
Layer 2: 프로젝트 상태 스냅샷  ← 주기적/이벤트 기반 스냅샷 저장
Layer 3: Claude Code 세션 복원 ← tmux 유지 또는 컨텍스트 재주입
```

---

## 1. 세션 생명주기

### 1.1 상태 전이 다이어그램

```
                  ┌─────────────────────────────────────────────┐
                  │               세션 생명주기                   │
                  └─────────────────────────────────────────────┘

  PM이 Set 생성
       │
       ▼
  ┌─────────┐   세션 시작 완료    ┌─────────┐
  │  생성중  │─────────────────→  │  활성   │
  │(CREATING)│                   │(ACTIVE) │
  └─────────┘                   └────┬────┘
                                      │
                    ┌─────────────────┼─────────────────┐
                    │                 │                   │
               PM 오프라인        PM 30분~2시간       명시적 중지
               30분 이내          이상 부재             요청
                    │                 │                   │
                    ▼                 ▼                   ▼
              ┌──────────┐     ┌──────────┐        ┌──────────┐
              │  유휴    │     │  슬립    │        │  종료중  │
              │  (IDLE)  │     │ (SLEEP)  │        │(STOPPING)│
              └────┬─────┘     └────┬─────┘        └────┬─────┘
                   │                │                    │
              PM 복귀           PM 복귀             스냅샷 저장
                   │          (스냅샷→재생성)             │
                   │                │                    ▼
                   └────────┬───────┘              ┌──────────┐
                            │                      │  종료됨  │
                            ▼                      │(STOPPED) │
                       ┌─────────┐                 └──────────┘
                       │  활성   │                       │
                       │(ACTIVE) │              PM 재접속 (복원 요청)
                       └─────────┘                       │
                                                         ▼
                                                   ┌─────────┐
                                                   │  생성중  │
                                                   │(CREATING)│
                                                   └─────────┘
```

### 1.2 상태별 정의

| 상태 | 설명 | 허용 동작 |
|---|---|---|
| `CREATING` | Claude Code 프로세스 기동 중 | 없음 (대기) |
| `ACTIVE` | PM 활성, 리더가 정상 응답 | 모든 작업 허용 |
| `IDLE` | PM 30분 이내 부재 | 진행 중 작업 계속, 의사결정 큐잉 |
| `SLEEP` | PM 30분~2시간 부재 | 현재 작업 마무리만, 신규 시작 금지 |
| `STOPPING` | 스냅샷 저장 후 프로세스 종료 중 | 없음 |
| `STOPPED` | 세션 종료됨 | 복원 요청만 가능 |

## 2. Layer 1: 대화 히스토리 (Firestore 자동 보존)

### 2.1 원리

모든 Council Room 메시지는 Firestore에 즉시 저장된다. 세션이 종료되어도 대화 내용은 영구 보존되므로, PM이 브라우저를 다시 열면 이전 대화를 그대로 확인할 수 있다.

```
PM 메시지 전송
      │
      ▼
Council Server
      │
      ├─→ Firestore: rooms/{roomId}/messages/{msgId} 저장 (영구)
      │
      └─→ Claude Code 세션으로 전달 (휘발성)
```

Firestore `onSnapshot` 리스너를 통해 클라이언트는 실시간으로 메시지를 수신한다. 오프라인 이후 복구 시에도 자동 동기화된다.

```typescript
// 클라이언트: Council Room 실시간 구독
import { collection, query, orderBy, onSnapshot } from 'firebase/firestore'

function subscribeToMessages(
  db: Firestore,
  projectId: string,
  roomId: string,
  onNewMessage: (msg: Message) => void
) {
  const messagesRef = collection(
    db,
    `projects/${projectId}/rooms/${roomId}/messages`
  )
  const q = query(messagesRef, orderBy('timestamp'))

  return onSnapshot(q, (snapshot) => {
    snapshot.docChanges().forEach((change) => {
      if (change.type === 'added') {
        onNewMessage(change.doc.data() as Message)
      }
    })
  })
}
```

### 2.2 Layer 1의 한계

대화 히스토리는 복구되지만, **Claude Code 세션 내 컨텍스트**는 복구되지 않는다. 즉, 리더는 이전 대화를 볼 수 없는 "빈 상태"로 재시작된다. Layer 2~3이 이를 보완한다.

## 3. Layer 2: 프로젝트 상태 스냅샷

### 3.1 스냅샷 데이터 구조

Firestore 저장 경로: `projects/{projectId}/snapshots/{snapshotId}`

```typescript
interface ProjectSnapshot {
  // 식별 정보
  snapshotId: string
  projectId: string
  createdAt: Timestamp

  // 스냅샷 생성 맥락
  trigger: SnapshotTrigger
  triggeredBy?: string             // 트리거 주체 (userId 또는 'system')

  // 프로젝트 요약
  summary: string                  // 자연어 요약 (LLM이 생성)
  phase: string                    // "Phase 1 — 백엔드 API 구현 중"

  // 태스크 현황
  completedTasks: CompletedTaskRef[]
  inProgressTasks: InProgressTaskRef[]
  blockedTasks: BlockedTaskRef[]

  // 주요 결정사항 (Council 대화에서 추출)
  decisions: DecisionRecord[]

  // Git 상태
  gitState: GitStateSnapshot

  // Set별 상태
  setStates: SetStateSnapshot[]

  // 최근 메시지 참조 (컨텍스트 재주입용)
  recentMessageIds: string[]       // 최근 30개 메시지 ID
  keyMessageIds: string[]          // 중요 결정이 담긴 메시지 ID
}

type SnapshotTrigger =
  | 'pr_merged'
  | 'task_completed'
  | 'task_status_changed'
  | 'periodic'                     // 30분 주기
  | 'manual'                       // PM 명시 요청
  | 'session_sleep'                // 2시간 부재 시
  | 'session_stop'                 // 세션 종료 전

interface CompletedTaskRef {
  taskId: string
  title: string
  completedAt: Timestamp
  setId: string
  prUrl?: string
}

interface InProgressTaskRef {
  taskId: string
  title: string
  setId: string
  setName: string
  progress: string                 // "70%"
  branch: string
  lastCommitMessage?: string
}

interface BlockedTaskRef {
  taskId: string
  title: string
  blockedReason: string
  blockedSince: Timestamp
}

interface DecisionRecord {
  content: string                  // "SimpleBroker 사용 (Redis는 2차)"
  decidedAt: Timestamp
  messageId: string                // 결정이 내려진 메시지
  participants: string[]           // 참여 Set ID들
}

interface GitStateSnapshot {
  mainBranch: string
  mainCommitCount: number
  mainLastCommitMessage: string
  openPRs: OpenPRRef[]
  branches: BranchStateRef[]
}

interface OpenPRRef {
  prId: string
  title: string
  number?: number
  sourceBranch: string
  targetBranch: string
  setId: string
  status: string
}

interface BranchStateRef {
  name: string
  setId: string
  aheadCount: number
  lastCommitMessage: string
}

interface SetStateSnapshot {
  setId: string
  setName: string
  status: 'idle' | 'working' | 'waiting' | 'done'
  currentTaskId?: string
  currentTaskTitle?: string
  worktreePath: string
  branch: string
}
```

### 3.2 스냅샷 생성 트리거

#### 이벤트 기반 트리거

```
┌──────────────────────────────────────────────────────┐
│               스냅샷 자동 생성 이벤트                  │
├──────────────────────────────────────────────────────┤
│                                                      │
│  PR 머지                                             │
│    git.onPRMerged() → createSnapshot('pr_merged')   │
│    이유: 코드베이스 상태가 변경되는 중요 시점           │
│                                                      │
│  태스크 완료                                          │
│    board.onTaskDone() → createSnapshot('task_completed') │
│    이유: 작업 단위가 완료된 명확한 체크포인트           │
│                                                      │
│  태스크 상태 변경 (주요 전환만)                        │
│    backlog→in_progress, in_progress→review          │
│    → createSnapshot('task_status_changed')          │
│                                                      │
│  PM 명시적 요청                                       │
│    PM: "상태 저장해줘" / "체크포인트 만들어"           │
│    → createSnapshot('manual')                       │
│                                                      │
│  세션 슬립 진입                                       │
│    pm.onAbsence(2h) → createSnapshot('session_sleep')│
│    이유: 세션 종료 전 마지막 상태 보존                  │
│                                                      │
│  세션 종료 직전                                       │
│    session.onStop() → createSnapshot('session_stop') │
│                                                      │
└──────────────────────────────────────────────────────┘
```

#### 주기적 트리거

```
세션 활성 상태일 때만 동작
  └── 30분마다 → createSnapshot('periodic')

단, 직전 이벤트 기반 스냅샷으로부터 10분 이내라면 스킵
(불필요한 중복 스냅샷 방지)
```

### 3.3 스냅샷 생성 구현

```typescript
// packages/server/src/session/SnapshotService.ts

export class SnapshotService {
  constructor(
    private firestore: Firestore,
    private taskService: TaskService,
    private gitService: GitService,
    private messageService: MessageService
  ) {}

  async createSnapshot(
    projectId: string,
    trigger: SnapshotTrigger,
    triggeredBy?: string
  ): Promise<ProjectSnapshot> {
    // 1. 병렬로 현재 상태 수집
    const [tasks, gitState, recentMessages, setStates] = await Promise.all([
      this.taskService.getAllTasks(projectId),
      this.gitService.getState(projectId),
      this.messageService.getRecent(projectId, 30),
      this.getSetStates(projectId),
    ])

    // 2. 태스크 분류
    const completedTasks = tasks
      .filter((t) => t.status === 'done')
      .map((t) => ({
        taskId: t.id,
        title: t.title,
        completedAt: t.updatedAt,
        setId: t.assignedSetId ?? '',
        prUrl: t.pullRequestUrl,
      }))

    const inProgressTasks = tasks
      .filter((t) => t.status === 'in_progress')
      .map((t) => ({
        taskId: t.id,
        title: t.title,
        setId: t.assignedSetId ?? '',
        setName: setStates.find((s) => s.setId === t.assignedSetId)?.setName ?? '',
        progress: '0%',          // Set에서 보고된 값
        branch: t.branch ?? '',
      }))

    // 3. 결정사항 추출 (키 메시지에서)
    const decisions = await this.extractDecisions(recentMessages)

    // 4. 요약 생성
    const summary = this.buildSummary(projectId, completedTasks, inProgressTasks, gitState)

    // 5. Firestore 저장
    const snapshot: ProjectSnapshot = {
      snapshotId: crypto.randomUUID(),
      projectId,
      createdAt: Timestamp.now(),
      trigger,
      triggeredBy,
      summary,
      phase: this.detectPhase(completedTasks, inProgressTasks),
      completedTasks,
      inProgressTasks,
      blockedTasks: tasks
        .filter((t) => t.status === 'blocked')
        .map((t) => ({
          taskId: t.id,
          title: t.title,
          blockedReason: '',
          blockedSince: t.updatedAt,
        })),
      decisions,
      gitState,
      setStates,
      recentMessageIds: recentMessages.map((m) => m.id),
      keyMessageIds: decisions.map((d) => d.messageId),
    }

    await setDoc(
      doc(this.firestore, `projects/${projectId}/snapshots/${snapshot.snapshotId}`),
      snapshot
    )

    // 6. 프로젝트 문서에 최신 스냅샷 ID 기록
    await updateDoc(doc(this.firestore, `projects/${projectId}`), {
      latestSnapshotId: snapshot.snapshotId,
      updatedAt: Timestamp.now(),
    })

    return snapshot
  }

  async getLatestSnapshot(projectId: string): Promise<ProjectSnapshot | null> {
    const projectDoc = await getDoc(doc(this.firestore, `projects/${projectId}`))
    const latestSnapshotId = projectDoc.data()?.latestSnapshotId
    if (!latestSnapshotId) return null

    const snapshotDoc = await getDoc(
      doc(this.firestore, `projects/${projectId}/snapshots/${latestSnapshotId}`)
    )
    return snapshotDoc.exists() ? (snapshotDoc.data() as ProjectSnapshot) : null
  }

  private buildSummary(
    projectId: string,
    completed: CompletedTaskRef[],
    inProgress: InProgressTaskRef[],
    gitState: GitStateSnapshot
  ): string {
    const completedCount = completed.length
    const inProgressCount = inProgress.length
    const openPRCount = gitState.openPRs.length

    return [
      `완료된 태스크 ${completedCount}개,`,
      `진행 중 ${inProgressCount}개.`,
      openPRCount > 0 ? `열린 PR ${openPRCount}개.` : '',
      `main 브랜치 ${gitState.mainCommitCount}커밋.`,
    ]
      .filter(Boolean)
      .join(' ')
  }

  private detectPhase(
    completed: CompletedTaskRef[],
    inProgress: InProgressTaskRef[]
  ): string {
    if (completed.length === 0 && inProgress.length === 0) return '초기 설계 단계'
    if (inProgress.length > 0) return `개발 진행 중 (완료 ${completed.length}개)`
    return `통합/검증 단계 (${completed.length}개 완료)`
  }

  private async extractDecisions(messages: Message[]): Promise<DecisionRecord[]> {
    // 시스템 메시지 또는 "결정", "합의", "채택" 키워드를 포함한 메시지에서 추출
    return messages
      .filter((m) =>
        m.senderType === 'leader' &&
        /결정|합의|채택|사용하기로|하기로/.test(m.content)
      )
      .map((m) => ({
        content: m.content.slice(0, 200),
        decidedAt: m.timestamp,
        messageId: m.id,
        participants: [],
      }))
  }
}
```

### 3.4 스냅샷 조회 전략

```typescript
// 스냅샷 조회 순서 (복원 시)
async function resolveSnapshot(projectId: string): Promise<ProjectSnapshot | null> {
  const snapshotService = new SnapshotService(...)

  // 1. 가장 최근 스냅샷 조회
  const latest = await snapshotService.getLatestSnapshot(projectId)
  if (latest) return latest

  // 2. 없으면 메시지에서 즉석 생성
  const generatedSnapshot = await snapshotService.createSnapshot(
    projectId,
    'manual',
    'system'
  )
  return generatedSnapshot
}
```

## 4. Layer 3: Claude Code 세션 복원

### 4.1 복원 전략 결정 흐름

```
PM이 프로젝트에 접속
         │
         ▼
   세션 상태 확인

                   │     즉석 스냅샷
```

### 4.2 전략 A: tmux 기반 세션 유지

PM이 없어도 Claude Code 프로세스를 서버에 살려두는 전략.

#### tmux 세션 명명 규칙

```
council-{projectId}-{setId}

예시:
  council-proj_abc123-set_arch    ← 아키텍처팀
  council-proj_abc123-set_back    ← 백엔드팀
  council-proj_abc123-set_front   ← 프론트팀
```

#### 구현

```typescript
// packages/server/src/session/TmuxSessionManager.ts

import { execSync, spawn } from 'child_process'

export class TmuxSessionManager {
  private sessionName(projectId: string, setId: string): string {
    return `council-${projectId}-${setId}`
  }

  /** 새 tmux 세션에서 Claude Code 시작 */
  async startSession(
    projectId: string,
    setId: string,
    worktreePath: string,
    systemPrompt: string
  ): Promise<{ pid: number; sessionName: string }> {
    const name = this.sessionName(projectId, setId)

    // 기존 세션이 있으면 종료
    this.killSession(name)

    // 새 tmux 세션 생성 (detached)
    // claude 프로세스는 worktree 경로에서 시작
    execSync(
      `tmux new-session -d -s ${name} -c ${worktreePath} ` +
      `"claude --system-prompt '${systemPrompt.replace(/'/g, "'\\''")}' " `
    )

    // PID 획득
    const pid = parseInt(
      execSync(`tmux list-panes -t ${name} -F '#{pane_pid}'`).toString().trim()
    )

    return { pid, sessionName: name }
  }

  /** 세션이 살아있는지 확인 */
  isAlive(projectId: string, setId: string): boolean {
    const name = this.sessionName(projectId, setId)
    try {
      execSync(`tmux has-session -t ${name}`, { stdio: 'ignore' })
      return true
    } catch {
      return false
    }
  }

  /** tmux 세션을 통해 메시지 전달 */
  sendMessage(projectId: string, setId: string, message: string): void {
    const name = this.sessionName(projectId, setId)
    // tmux send-keys로 입력 (실제 구현에서는 stdio pipe 사용)
    const escaped = message.replace(/'/g, "'\\''")
    execSync(`tmux send-keys -t ${name} '${escaped}' Enter`)
  }

  /** 세션 강제 종료 */
  killSession(nameOrId: string): void {
    try {
      execSync(`tmux kill-session -t ${nameOrId}`, { stdio: 'ignore' })
    } catch {
      // 세션이 없으면 무시
    }
  }

  /** 모든 Council 세션 목록 */
  listSessions(): string[] {
    try {
      const output = execSync(
        `tmux list-sessions -F '#{session_name}' 2>/dev/null`
      ).toString()
      return output
        .split('\n')
        .filter((s) => s.startsWith('council-'))
    } catch {
      return []
    }
  }
}
```

```typescript
// packages/server/src/session/SessionRestorer.ts

export class SessionRestorer {
  constructor(
    private tmuxManager: TmuxSessionManager,
    private snapshotService: SnapshotService,
    private contextInjector: ContextInjector,
    private messageService: MessageService
  ) {}

  async restoreSession(
    projectId: string,
    setId: string,
    setConfig: SetConfig
  ): Promise<ClaudeSession> {
    // 1. 기존 세션 살아있는지 확인 (전략 A)
    const isAlive = this.tmuxManager.isAlive(projectId, setId)
    if (isAlive) {
      console.log(`[Session] 기존 세션 재연결: ${setId}`)
      return this.reconnectExistingSession(projectId, setId)
    }

    // 2. 세션 죽음 → 전략 B
    console.log(`[Session] 세션 재생성 시작: ${setId}`)
    return this.recreateWithContext(projectId, setId, setConfig)
  }

  private async recreateWithContext(
    projectId: string,
    setId: string,
    setConfig: SetConfig
  ): Promise<ClaudeSession> {
    // 2-1. 스냅샷 조회 (없으면 즉석 생성)
    let snapshot = await this.snapshotService.getLatestSnapshot(projectId)
    if (!snapshot) {
      console.log('[Session] 스냅샷 없음 — 메시지에서 즉석 생성')
      snapshot = await this.snapshotService.createSnapshot(projectId, 'manual', 'system')
    }

    // 2-2. 최근 메시지 조회
    const recentMessages = await this.messageService.getRecent(projectId, 30)

    // 2-3. 컨텍스트 주입 프롬프트 빌드
    const currentBranch = setConfig.branch ?? `set-${setId}/main`
    const restorationPrompt = buildRestorationPrompt(
      setConfig.role,
      snapshot,
      recentMessages,
      currentBranch
    )

    // 2-4. 새 tmux 세션 시작
    const { pid, sessionName } = await this.tmuxManager.startSession(
      projectId,
      setId,
      setConfig.worktreePath,
      restorationPrompt
    )

    return {
      sessionId: crypto.randomUUID(),
      projectId,
      setId,
      status: 'ACTIVE',
      tmuxSessionName: sessionName,
      pid,
      createdAt: new Date(),
      lastActiveAt: new Date(),
      lastHeartbeatAt: new Date(),
      worktreePath: setConfig.worktreePath,
      snapshotId: snapshot.snapshotId,
    }
  }

  private reconnectExistingSession(
    projectId: string,
    setId: string
  ): ClaudeSession {
    const sessionName = `council-${projectId}-${setId}`
    return {
      sessionId: crypto.randomUUID(),
      projectId,
      setId,
      status: 'ACTIVE',
      tmuxSessionName: sessionName,
      createdAt: new Date(),
      lastActiveAt: new Date(),
      lastHeartbeatAt: new Date(),
      worktreePath: '',           // 기존 세션에서 유지
    }
  }
}
```

### 5.2 PM 활동 감지 구현

```typescript
// packages/server/src/session/PMPresenceMonitor.ts

export class PMPresenceMonitor {
  private lastSeenAt = new Map<string, Date>()   // projectId → 마지막 활동 시각
  private timers = new Map<string, NodeJS.Timeout>()

  /** PM 활동 기록 (메시지 전송, API 호출 등) */
  recordActivity(projectId: string): void {
    this.lastSeenAt.set(projectId, new Date())
    this.resetTimer(projectId)
  }

  private resetTimer(projectId: string): void {
    const existing = this.timers.get(projectId)
    if (existing) clearTimeout(existing)

    // 30분 후 IDLE 체크
    const timer = setTimeout(() => {
      this.onAbsenceThreshold(projectId, '30min')
    }, 30 * 60 * 1000)

    this.timers.set(projectId, timer)
  }

  private async onAbsenceThreshold(
    projectId: string,
    threshold: '30min' | '2hour'
  ): Promise<void> {
    const lastSeen = this.lastSeenAt.get(projectId)
    if (!lastSeen) return

    const absenceMs = Date.now() - lastSeen.getTime()
    const absenceMin = absenceMs / 60000

    if (absenceMin < 30) return

    if (absenceMin >= 120) {
      // 2시간 초과 → 세션 종료
      await this.sessionManager.transitionToSleep(projectId)
    } else if (absenceMin >= 30) {
      // 30분~2시간 → IDLE → SLEEP
      await this.sessionManager.transitionToIdle(projectId)

      // 2시간 타이머 재설정
      const sleepTimer = setTimeout(() => {
        this.onAbsenceThreshold(projectId, '2hour')
      }, (120 - absenceMin) * 60 * 1000)

      this.timers.set(`${projectId}_sleep`, sleepTimer)
    }
  }

  getAbsenceDuration(projectId: string): number {
    const lastSeen = this.lastSeenAt.get(projectId)
    if (!lastSeen) return Infinity
    return Date.now() - lastSeen.getTime()
  }
}
```

```typescript
// packages/server/src/ws/HeartbeatHandler.ts

export class HeartbeatHandler {
  private readonly HEARTBEAT_INTERVAL_ACTIVE = 30_000   // 30초
  private readonly HEARTBEAT_INTERVAL_SLEEP  = 300_000  // 5분

  handlePing(ws: WebSocket, payload: HeartbeatPing): void {
    const { projectId, userId } = payload

    // PM 활동 기록
    this.presenceMonitor.recordActivity(projectId)

    // 세션 상태 확인
    const sessions = this.sessionManager.getSessionsForProject(projectId)
    const sessionStatuses = sessions.map((s) => ({
      setId: s.setId,
      status: s.status,
      isAlive: this.tmuxManager.isAlive(projectId, s.setId),
    }))

    // 죽은 세션 감지 → 복원 이벤트 발행
    const deadSessions = sessionStatuses.filter((s) => !s.isAlive)
    if (deadSessions.length > 0) {
      ws.send(JSON.stringify({
        type: 'session_dead',
        deadSetIds: deadSessions.map((s) => s.setId),
      }))
    } else {
      ws.send(JSON.stringify({
        type: 'pong',
        timestamp: Date.now(),
        sessions: sessionStatuses,
      }))
    }
  }
}

  userId: string
  timestamp: number
}
```

### 6.3 Firestore 오프라인 대응

Firestore SDK는 오프라인 시 쓰기를 로컬에 큐잉하고, 연결 복구 시 자동 동기화한다. 서버 측에서 추가적으로 보장한다.

```typescript
// packages/server/src/firebase/FirestoreWithRetry.ts

export class FirestoreWithRetry {
  private readonly MAX_RETRIES = 5
  private readonly RETRY_DELAY_MS = 1000

  async writeWithRetry<T>(
    operation: () => Promise<T>,
    context: string
  ): Promise<T> {
    let lastError: Error | null = null

    for (let attempt = 1; attempt <= this.MAX_RETRIES; attempt++) {
      try {
        return await operation()
      } catch (err) {
        lastError = err as Error
        console.warn(`[Firestore] ${context} 실패 (시도 ${attempt}/${this.MAX_RETRIES}):`, err)

        if (attempt < this.MAX_RETRIES) {
          await this.sleep(this.RETRY_DELAY_MS * attempt)  // 지수 백오프
        }
      }
    }

    // 최종 실패 시 로컬 파일에 백업
    await this.localBackup.save(context, lastError)
    throw lastError
  }

  private sleep(ms: number): Promise<void> {
    return new Promise((resolve) => setTimeout(resolve, ms))
  }
}
```

| 평균 복원 소요시간 | 전략 B 실행 시간 | > 30초 시 경고 |

| 스냅샷 생성 실패율 | 실패 / 전체 시도 | > 5% 시 알림 |
| tmux 세션 수 | 서버 내 활성 tmux 세션 | > 20 시 리소스 경고 |
