## 1. 설계 원칙 요약

```
1. 대화가 중심이다
   Council Room 채팅이 항상 주인공. 보드·Git·로그는 채팅을 보조.

2. 정보는 숨기지 않되, 단계적으로 드러낸다
   기본 뷰는 채팅. 나머지는 탭/스와이프로 도달.
   "3탭 이내에 모든 정보 접근 가능" 원칙.

3. 상태는 항상 보인다
   Set 상태(작업 중/대기/완료)는 어떤 화면에서든 확인 가능.
   새 메시지·PR·태스크 변경은 실시간 뱃지.

4. 맥락 전환을 최소화한다
   채팅에서 태스크 등록·PR 확인이 인라인으로 가능.
   별도 페이지 이동 없이 핵심 작업 수행.
```

## 3. 로그인 화면 `/login`

### 목적

Firebase Auth를 통한 사용자 인증. Google 또는 GitHub OAuth 소셜 로그인만 지원 (패스워드 로그인 없음).

### 진입 경로

- 앱 최초 접근 (미인증 상태)
- 보호된 라우트 접근 시 자동 리다이렉트
- 로그아웃 후

### 주요 인터랙션

1. Google 로그인 버튼 클릭 → Firebase Auth Google OAuth 팝업
2. GitHub 로그인 버튼 클릭 → Firebase Auth GitHub OAuth 팝업
3. 인증 성공 → 기존 방문 경로 또는 `/` 리다이렉트
4. 인증 실패 → 에러 메시지 인라인 표시

### Firebase Auth 플로우

```
사용자 클릭 (Google/GitHub)
    │
    ▼
signInWithPopup(auth, provider)
    │
    ├── 성공: UserCredential 수신
    │        ├── users/{uid} 문서 존재 여부 확인
    │        │     ├── 있음: 마지막 로그인 시간 업데이트
    │        │     └── 없음: 신규 사용자 문서 생성
    │        └── redirectTo 또는 '/'로 이동
    │
    └── 실패: AuthError 코드별 메시지 표시
             ├── auth/popup-closed-by-user → 안내 없음 (조용히 처리)
             ├── auth/popup-blocked → "팝업이 차단되었습니다. 허용해주세요."
             └── 기타 → "로그인에 실패했습니다. 다시 시도해주세요."
```

### 상태별 분기

| 기본 | 로그인 버튼 활성화 |
| 로딩 중 | 버튼 비활성화 + 스피너, "로그인 중..." |
| 성공 | 페이드아웃 후 대시보드로 전환 |
| 에러 | 버튼 아래 빨간 에러 메시지, 버튼 재활성화 |

## 4. 대시보드 `/`

### 목적

사용자가 소유한 프로젝트 목록 표시 및 새 프로젝트 생성 진입점. 최근 활동 피드로 빠른 복귀 지원.

### 진입 경로

- 로그인 성공 직후
- 헤더 로고 클릭
- 사이드 패널 홈 아이콘 클릭

### 주요 인터랙션

1. 프로젝트 카드 클릭 → `/p/{id}` (Council Room) 이동
2. "+ 새 프로젝트" 버튼 클릭 → `/new` 이동
3. 프로젝트 카드 더보기(···) 메뉴 → 아카이브 / 삭제
4. 최근 활동 항목 클릭 → 해당 Council Room + 메시지 위치로 스크롤

### 상태별 분기

| 상태 | 화면 표현 |
|---|---|
| 로딩 중 | 카드 3개 스켈레톤 애니메이션 |
| 빈 상태 (프로젝트 없음) | 일러스트 + "첫 프로젝트를 만들어보세요" + CTA 버튼 |
| 정상 | 프로젝트 카드 그리드 + 최근 활동 피드 |
| 에러 | "데이터를 불러오지 못했습니다." + 재시도 버튼 |

## 5. 프로젝트 생성 `/new`

### 목적

3가지 프로젝트 유형 선택 후 단계별 위저드(wizard) 폼으로 프로젝트 설정. 완료 시 Council Room 자동 생성.

### 진입 경로

- 대시보드 "+ 새 프로젝트" 버튼
- 대시보드 빈 상태 CTA 버튼

### 주요 인터랙션

1. 유형 선택 카드 클릭 → 해당 유형 Step 2로 진행
2. 각 단계 "다음" / "이전" 버튼으로 네비게이션
3. 최종 "생성" 클릭 → 서버에 프로젝트 생성 요청 → 완료 시 `/p/{id}` 이동
4. 상단 스텝 인디케이터 클릭으로 이전 단계 복귀 가능

### 3가지 프로젝트 유형

| 유형 | 아이콘 | 설명 |
|---|---|---|
| A. 신규 프로젝트 | ✨ | 빈 Git 저장소에서 새로 시작 |
| B. 기존 프로젝트 | 🔗 | 이미 있는 GitHub repo에 기능 추가 |
| C. 분석 프로젝트 | 🔍 | 코드 수정 없이 분석·리포트 전용 |

### 단계별 폼 구조

#### 유형 A — 신규 프로젝트

```
Step 1: 유형 선택
Step 2: 기본 정보  → 프로젝트명, 설명, 기술 스택 태그
Step 3: Set 구성  → 자동 추천 또는 수동 Set 추가 (이름, 역할, 팀원 수, 색상)
Step 4: 확인      → 설정 요약 미리보기 + "생성" 버튼
```

#### 유형 B — 기존 프로젝트

```
Step 1: 유형 선택
Step 2: 저장소 연결 → GitHub repo URL 또는 로컬 경로 입력 + GitHub PAT (선택)
Step 3: 기본 정보  → 프로젝트명, 설명 (repo 정보에서 자동 채워짐)
Step 4: Set 구성  → 기존 코드 분석 후 자동 추천
Step 5: 확인      → 요약 + "생성"
```

#### 유형 C — 분석 프로젝트

```
Step 1: 유형 선택
Step 2: 저장소 연결 → 분석 대상 repo URL (읽기 전용)
Step 3: 기본 정보  → 프로젝트명, 분석 목표 (보안/성능/품질 등)
Step 4: Set 구성  → 분석 전용 Set 구성 (코드 수정 불가 플래그)
Step 5: 확인      → 요약 + "생성"
```

### 상태별 분기

| 상태 | 화면 표현 |
|---|---|
| Step 이동 중 | 스텝 인디케이터 갱신, 폼 슬라이드 전환 |
| 저장소 검증 중 | URL 입력 필드 아래 "저장소 확인 중..." 스피너 |
| 저장소 오류 | "저장소를 찾을 수 없습니다." 인라인 에러 |
| 생성 중 | 전체 오버레이 스피너 + "프로젝트를 생성하고 있습니다..." |
| 생성 완료 | 체크 애니메이션 → 자동으로 `/p/{id}` 이동 |
| 생성 실패 | 에러 토스트 + 재시도 버튼 |

## 9. 설정 `/settings`

### 목적

사용자 프로필 관리, Anthropic API 키 등록 (BYOK), GitHub 토큰, UI 테마 설정.

### 진입 경로

- 헤더 [설정] 버튼 클릭
- 모바일 하단 탭 [설정] 클릭

### 섹션 구성

1. **프로필** — 표시 이름, 아바타 (Firebase Auth에서 가져옴, 수정 가능)
2. **API 키** — Anthropic API 키 또는 Max 플랜 인증 파일
3. **GitHub 연동** — GitHub Personal Access Token (PR 생성·머지용)
4. **테마** — 다크/라이트/시스템 따름
5. **계정** — 로그아웃, 계정 삭제

### API 키 입력 규칙

- 키 입력 필드: `sk-ant-...` 형식 자동 감지
- 저장 시 서버로 전송 → AES-256 암호화 → Firestore `users/{uid}.apiKeyEncrypted` 저장
- 화면에는 마스킹 표시: `sk-ant-••••••••••••••••••••••`
- "키 검증" 버튼 → 서버에서 간단한 API 호출 테스트 → 결과 표시

### 상태별 분기

| 상태 | 화면 표현 |
|---|---|
| 저장 중 | 버튼 스피너, 필드 비활성화 |
| 저장 성공 | 초록 "저장되었습니다." 토스트 |
| 키 검증 중 | "API 키 확인 중..." 인라인 메시지 |
| 키 유효 | 초록 체크 + "API 키가 유효합니다." |
| 키 무효 | 빨간 경고 + "유효하지 않은 API 키입니다." |
| 저장 실패 | 빨간 에러 토스트 |

| QA | 노랑 | `#EAB308` | `yellow-500` |
| DevOps | 주황 | `#F97316` | `orange-500` |
| 보안 | 빨강 | `#EF4444` | `red-500` |
| 디자인 | 핑크 | `#EC4899` | `pink-500` |
| 데이터 | 시안 | `#06B6D4` | `cyan-500` |

### 10.2 상태 색상

| 상태 | 색상 | 용도 |
|---|---|---|
| `working` | 파랑 (파동 애니메이션) | Set이 코드 작업 중 |
| `waiting` | 노랑 | PM 또는 다른 Set 대기 |
| `done` | 초록 | 현재 태스크 완료 |
| `idle` | 회색 | 유휴 상태 |
| `error` | 빨강 | 오류 발생 |

### 10.3 타이포그래피

| 용도 | 폰트 | 크기 |
|---|---|---|
| 채팅 메시지 | system-ui | 14px |
| Set 내부 로그 | JetBrains Mono | 13px |
| 코드 블록 | JetBrains Mono | 13px |
| 메타 정보 (시간, 상태) | system-ui | 12px |
| UI 레이블 | system-ui | 14px |
| 헤더 | system-ui | 16px / 600 weight |

### 10.4 반응형 브레이크포인트

| 구분 | 범위 | 레이아웃 |
|---|---|---|
| 모바일 | `< 768px` | 풀스크린 + 바텀 탭 바 |
| 태블릿 | `768px ~ 1023px` | 2패널 (Left 햄버거) |
| 데스크톱 | `≥ 1024px` | 3패널 고정 |

### 10.5 공통 인터랙션 패턴

| 패턴 | 구현 | 적용 화면 |
|---|---|---|
| Skeleton Loading | shimmer 애니메이션 | 모든 데이터 로딩 |
| Optimistic Update | 로컬 상태 즉시 반영 | 메시지 전송, 태스크 이동 |
| Pull-to-refresh | 모바일 스와이프 아래 | Git 현황, 보드 |
| Toast 알림 | 하단 우측, 3초 자동 닫힘 | 저장 성공/실패, 머지 결과 |
| Infinite Scroll | 위로 스크롤 시 이전 메시지 | Council Room 채팅 |
| Keyboard Shortcut | `Enter` 전송, `Esc` 취소 | 메시지 입력, 다이얼로그 |

### 10.6 빈 상태 (Empty State) 표준

```
┌─────────────────────────┐
│         [아이콘]          │
│                         │
│   [제목: 없음 상태 설명] │
│                         │
│   [부제: 방법 안내]       │
│                         │
│     [ CTA 버튼 ]         │
└─────────────────────────┘
```

모든 목록 화면은 빈 상태 정의 필수. CTA 버튼은 가장 자연스러운 다음 액션 연결.

### 10.7 에러 상태 표준

- **인라인 에러**: 입력 필드 아래, 빨간 텍스트 + 아이콘
- **섹션 에러**: "데이터를 불러오지 못했습니다." + [다시 시도] 버튼
- **전체 에러**: 에러 페이지 (뒤로 가기 링크 포함)
- **토스트 에러**: 3~5초 표시, 빨간 배경, X 닫기 버튼
