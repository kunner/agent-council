## 1. 브레이크포인트 정의

### 1.1 구간 분류

| 이름 | 구간 | 대상 기기 |
|---|---|---|
| **mobile** | `< 768px` | 스마트폰 (세로 방향 기준) |
| **tablet** | `768px ~ 1023px` | 태블릿, 소형 노트북 |
| **desktop** | `≥ 1024px` | 노트북, 데스크톱 모니터 |

### 1.2 Tailwind 커스텀 설정

```ts
// tailwind.config.ts
export default {
  theme: {
    screens: {
      // mobile: 기본값 (0px~767px) — 접두사 없음
      'md': '768px',   // tablet 시작
      'lg': '1024px',  // desktop 시작
      // 필요 시 xl: '1280px', 2xl: '1536px' 추가
    },
  },
}
```

**사용 패턴 — Mobile First:**
```tsx
// 모바일 기본 → 태블릿 오버라이드 → 데스크톱 오버라이드
<div className="flex-col md:flex-row lg:flex-row">
```

## 2. 핵심 원칙

### 2.1 "정보는 숨기지 않되 단계적으로 드러낸다"

모든 화면 크기에서 모든 정보에 접근할 수 있어야 한다. 단, **기본 뷰**는 가장 중요한 정보(채팅)만 표시하고, 나머지는 탭·스와이프·드로어로 점진적으로 노출한다.

```
데스크톱: 보드 + 채팅 + Git 동시 표시
          ↓ 화면이 좁아짐
태블릿:   채팅 + 컨텍스트 패널 (보드/Git 탭으로 전환)
          ↓ 화면이 더 좁아짐
모바일:   채팅 풀스크린 (보드/Git는 바텀탭 1탭으로 접근)
```

어떤 기기에서도 정보가 **완전히 차단되지 않는다**. 숨기는 게 아니라 이동하는 것이다.

### 2.2 "3탭 이내 모든 정보 접근"

임의의 화면에서 임의의 정보까지 도달하는 데 탭(또는 스와이프) 3회를 초과하지 않는다.

| 정보 | 데스크톱 | 태블릿 | 모바일 |
|---|---|---|---|
| 채팅 메시지 | 0 (항상 보임) | 0 (항상 보임) | 0 (기본 화면) |
| 칸반 보드 | 1 (Right 탭) | 1 (Right 탭) | 1 (바텀탭) |
| Git 상태 | 1 (Right 탭) | 1 (Right 탭) | 1 (바텀탭) |
| Set 내부 로그 | 1 (인라인 펼치기) | 1 (인라인 펼치기) | 1 (탭하여 펼치기) |
| 프로젝트 목록 | 0 (Left 패널) | 1 (햄버거 메뉴) | 1 (햄버거 메뉴) |
| 설정 | 1 (상단 아이콘) | 1 (상단 아이콘) | 1 (바텀탭) |

최대 탭 수: **1** (모든 항목이 1탭 이내).

### 2.3 상태는 항상 보인다

Set 상태(작업 중/대기/완료)와 새 메시지 뱃지는 어떤 화면에서든 시야에 있어야 한다.

- **데스크톱**: Left 패널에 Set별 상태 목록 상시 표시
- **태블릿**: 상단 바에 Set 이름 + 상태 아이콘 요약
- **모바일**: 상단 바에 컬러 도트만 표시 (색상 = Set 색상, 펄스 애니메이션 = 작업 중)

### 3.3 패널 크기 조절 (드래그 리사이즈)

데스크톱에서 Left/Right 패널 경계를 드래그하여 너비를 조정할 수 있다.

```
드래그 리사이즈 제약:
  Left Panel:  최소 200px ~ 최대 400px  (기본 280px)
  Right Panel: 최소 240px ~ 최대 480px  (기본 320px)
  Center Panel: 최소 400px (항상 보장)
```

```tsx
// react-resizable-panels 라이브러리 활용
import {
  PanelGroup,
  Panel,
  PanelResizeHandle,
} from 'react-resizable-panels'

// 데스크톱 전용 (lg:flex)
<PanelGroup direction="horizontal" className="hidden lg:flex h-full">
  <Panel
    defaultSize={20}   // 전체 너비의 20%
    minSize={15}       // 최소 15%
    maxSize={30}       // 최대 30%
    className="overflow-y-auto"
  >
    <LeftPanel />
  </Panel>

  {/* 드래그 핸들 — 4px 너비, hover 시 강조 */}
  <PanelResizeHandle className="w-1 bg-border hover:bg-primary/30
                                 transition-colors cursor-col-resize" />

  <Panel minSize={40} className="flex flex-col overflow-hidden">
    <CenterPanel />
  </Panel>

  <PanelResizeHandle className="w-1 bg-border hover:bg-primary/30
                                 transition-colors cursor-col-resize" />

  <Panel
    defaultSize={23}
    minSize={18}
    maxSize={35}
    className="overflow-y-auto"
  >
    <RightPanel />
  </Panel>
</PanelGroup>
```

## 4. 컴포넌트별 반응형 동작

### 4.1 채팅 메시지

메시지 자체의 구조는 모든 브레이크포인트에서 동일하다. 변하는 것은 **최대 너비와 좌우 패딩**뿐이다.

```
데스크톱 (Center Panel 600px+):  메시지 최대 너비 85%
태블릿 (Center Panel 400px+):    메시지 최대 너비 90%
모바일 (풀스크린):                메시지 최대 너비 92%
```

```tsx
// 메시지 버블 너비 조정
<div className="max-w-[92%] md:max-w-[90%] lg:max-w-[85%] ...">
  {message.content}
</div>
```

**메시지 영역 패딩:**
```tsx
<div className="px-3 py-2 md:px-4 md:py-3 lg:px-6 lg:py-3">
  <MessageBubble />
</div>
```

**Set 내부 로그 접기/펼치기:**
```tsx
// 모바일에서는 기본값 접힘, 태블릿/데스크톱은 기본값 펼침

const [expanded, setExpanded] = useState(defaultExpanded)
```

#### 모바일: 풀스크린 단일 열 + 상단 필터

```
┌─────────────────────────┐
│  [Backlog ▼]  필터: 전체▼ │  ← 드롭다운으로 열 전환
├─────────────────────────┤
│  ┌─────────────────────┐│
│  │ 파일 전송             ││
│  │ 미할당 | Medium       ││
│  └─────────────────────┘│
│  ┌─────────────────────┐│
│  │ 읽음 표시             ││
│  └─────────────────────┘│
│                          │
│  ← 스와이프: 다음 열(Working) →  │
└─────────────────────────┘
```

스와이프로 열 전환 시 상단 필터 드롭다운도 연동하여 현재 열 이름을 표시한다.

```tsx
// 모바일: 스와이프 가능한 단일 열 뷰
const [activeColumn, setActiveColumn] = useState(0)

<div className="flex md:hidden flex-col h-full">
  {/* 상단: 열 선택 드롭다운 */}
  <div className="px-4 py-2 flex items-center gap-2 border-b">
    <select
      value={activeColumn}
      onChange={e => setActiveColumn(Number(e.target.value))}
      className="font-semibold bg-transparent"
    >
      {columns.map((col, i) => (
        <option key={col.id} value={i}>
          {col.name} ({col.tasks.length})
        </option>
      ))}
    </select>
    <span className="ml-auto text-sm text-muted-foreground">
      필터: 전체
    </span>
  </div>

  {/* 스와이프 컨테이너 */}
  <div
    className="flex-1 overflow-hidden"
    onTouchStart={handleTouchStart}
    onTouchEnd={handleTouchEnd}
  >
    <TaskList tasks={columns[activeColumn].tasks} />
  </div>
</div>
```

**스와이프 핸들러:**
```tsx
const SWIPE_THRESHOLD = 60  // px

const handleTouchStart = (e: React.TouchEvent) => {
  touchStartX.current = e.touches[0].clientX
}

const handleTouchEnd = (e: React.TouchEvent) => {
  const delta = touchStartX.current - e.changedTouches[0].clientX
  if (Math.abs(delta) < SWIPE_THRESHOLD) return

  if (delta > 0) {
    // 왼쪽으로 스와이프 → 다음 열
    setActiveColumn(prev => Math.min(prev + 1, columns.length - 1))
  } else {
    // 오른쪽으로 스와이프 → 이전 열
    setActiveColumn(prev => Math.max(prev - 1, 0))
  }
}
```

### 4.3 Git 패널

#### 데스크톱/태블릿: Right Panel 탭

```
Right Panel 내 탭 방식으로 표시:
[보드] [Git] [로그]  ← 탭 헤더
─────────────────────
  Git 내용 (스크롤)
```

```
[  Backlog  ]  [ Working ◀▶ ]  [  Review  ]
```

### 5.2 롱프레스 (메시지 컨텍스트 메뉴)

PC에서는 우클릭으로 표시되는 컨텍스트 메뉴를 모바일/태블릿에서는 롱프레스(500ms)로 트리거한다.

```
메시지 롱프레스 (500ms 유지)
         ↓
  ┌─────────────────┐
  │  📋 태스크로 등록  │
  │  🔗 링크 복사     │
  │  ↩️  답글         │
  │  📌 고정          │
  └─────────────────┘
```

```tsx
// 롱프레스 커스텀 훅
const LONG_PRESS_DURATION = 500  // ms

const useLongPress = (onLongPress: () => void) => {
  const timerRef = useRef<ReturnType<typeof setTimeout> | null>(null)

  const start = (e: React.TouchEvent) => {
    e.preventDefault()  // 텍스트 선택 방지
    timerRef.current = setTimeout(onLongPress, LONG_PRESS_DURATION)
  }

  const cancel = () => {
    if (timerRef.current) clearTimeout(timerRef.current)
  }

  return {
    onTouchStart: start,
    onTouchEnd: cancel,
    onTouchMove: cancel,  // 스크롤 시 취소
  }
}

// 메시지 컴포넌트에 적용
const longPressProps = useLongPress(() => setContextMenuOpen(true))

<div {...longPressProps} className="message-bubble">
  {message.content}
</div>
```

**햅틱 피드백 (지원 기기):**
```tsx
const triggerHaptic = () => {
```

## 6. 성능 고려사항

### 6.1 메시지 가상화 (모바일)

활성 Council Room에는 수백~수천 개의 메시지가 쌓일 수 있다. 모바일에서 DOM에 전부 렌더링하면 스크롤 성능이 저하된다.

**임계값:**
- 메시지 수 < 100: 가상화 불필요
- 메시지 수 ≥ 100: 가상화 적용 (모든 기기)
- 모바일에서는 50개부터 적용

```tsx
// @tanstack/react-virtual 활용
import { useVirtualizer } from '@tanstack/react-virtual'

const ChatMessages = ({ messages }: { messages: Message[] }) => {
  const parentRef = useRef<HTMLDivElement>(null)
  const isMobile = useBreakpoint('md') === false

  // 모바일은 50개부터, 데스크톱은 100개부터 가상화
  const shouldVirtualize = messages.length > (isMobile ? 50 : 100)

  const virtualizer = useVirtualizer({
    count: shouldVirtualize ? messages.length : 0,
    getScrollElement: () => parentRef.current,
    estimateSize: () => 80,  // 메시지 평균 높이 추정
    overscan: 5,             // 뷰포트 밖 5개씩 미리 렌더
  })

  if (!shouldVirtualize) {
    // 가상화 없이 전부 렌더
    return (
      <div ref={parentRef} className="overflow-y-auto h-full">
        {messages.map(msg => <MessageBubble key={msg.id} message={msg} />)}
      </div>
    )
  }

  return (
    <div ref={parentRef} className="overflow-y-auto h-full">
      <div style={{ height: virtualizer.getTotalSize() + 'px', position: 'relative' }}>
        {virtualizer.getVirtualItems().map(item => (
          <div
            key={item.key}
            style={{
              position: 'absolute',
              top: item.start + 'px',
              width: '100%',
            }}
          >
            <MessageBubble message={messages[item.index]} />
          </div>
        ))}
      </div>
    </div>
  )
}
```

### 6.2 이미지/아티팩트 지연 로딩

메시지에 첨부된 코드 아티팩트 미리보기와 이미지는 뷰포트 진입 시점에 로드한다.

```tsx
// Intersection Observer 기반 지연 로딩
const LazyArtifactPreview = ({ artifact }: { artifact: Artifact }) => {
  const ref = useRef<HTMLDivElement>(null)
  const [visible, setVisible] = useState(false)

  useEffect(() => {
    const observer = new IntersectionObserver(
      ([entry]) => { if (entry.isIntersecting) setVisible(true) },
      { rootMargin: '200px' }  // 200px 전에 미리 로드 시작
    )
    if (ref.current) observer.observe(ref.current)
    return () => observer.disconnect()
  }, [])

  return (
    <div ref={ref} className="min-h-[60px]">
      {visible
        ? <ArtifactContent artifact={artifact} />
        : <ArtifactSkeleton />
      }
    </div>
  )
}
```

### 6.3 Shiki 코드 하이라이팅 — 모바일 최적화

코드 블록이 많을 때 Shiki의 동기 렌더링이 메인 스레드를 블록할 수 있다.

```tsx
// 모바일에서 코드 블록 지연 하이라이팅
const CodeBlock = ({ code, lang }: { code: string; lang: string }) => {
  const isMobile = useBreakpoint('md') === false
  const [highlighted, setHighlighted] = useState<string | null>(null)

  useEffect(() => {
    // 모바일에서는 idle 콜백으로 하이라이팅 지연
    const run = () => shiki.codeToHtml(code, { lang }).then(setHighlighted)

    if (isMobile && 'requestIdleCallback' in window) {
      requestIdleCallback(run, { timeout: 2000 })
    } else {
      run()
    }
  }, [code, lang])

  if (!highlighted) {
    return <pre className="font-mono text-sm p-3 bg-muted rounded">{code}</pre>
  }

  return <div dangerouslySetInnerHTML={{ __html: highlighted }} />
}
```

### 6.4 칸반 드래그앤드롭 — 모바일 비활성화

```tsx
const TaskCard = ({ task }: { task: Task }) => {
```

### 7.3 브레이크포인트 감지 훅

```tsx
// hooks/useBreakpoint.ts
import { useState, useEffect } from 'react'

type Breakpoint = 'mobile' | 'tablet' | 'desktop'

export const useBreakpoint = (): Breakpoint => {
  const [bp, setBp] = useState<Breakpoint>('desktop')

  useEffect(() => {
    const update = () => {
      const w = window.innerWidth
      if (w < 768) setBp('mobile')
      else if (w < 1024) setBp('tablet')
      else setBp('desktop')
    }

    update()
    window.addEventListener('resize', update)
    return () => window.removeEventListener('resize', update)
  }, [])

  return bp
}

// 사용 예
const bp = useBreakpoint()
const isMobile = bp === 'mobile'
const isDesktop = bp === 'desktop'
```
