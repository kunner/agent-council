## 1. 디자인 원칙

Agent Council은 AI 에이전트 팀을 실시간으로 오케스트레이션하는 **개발 도구**다. 일반적인 소비자 앱과는 달리, 사용자(PM, 개발자)는 긴 시간 동안 화면을 응시하며 밀도 높은 정보를 처리한다.

### 1.1 정보 밀도 우선

개발 도구는 미니멀리즘을 위해 정보를 희생하지 않는다.

- 메시지 버블에 발신자, 타임스탬프, 상태를 한 화면에 모두 표시한다
- 컴팩트한 간격(padding, margin)으로 한 화면에 더 많은 메시지를 표시한다
- 패널 간 경계는 명확하게, 공백 낭비는 최소화한다
- 단, 과밀은 피한다 — 줄간격과 그룹 간격으로 시각적 호흡을 유지한다

### 1.2 가독성 최우선

채팅 메시지, 코드 로그, 시스템 알림이 혼재하는 환경에서 빠른 스캔이 가능해야 한다.

- 메시지 타입별 시각적 계층(색상, 폰트 크기, 들여쓰기)을 일관되게 적용한다
- 코드/로그는 모노스페이스 폰트, 일반 텍스트는 시스템 폰트로 구분한다
- 발신자 컬러(Set 컬러)를 통해 누가 말하는지 텍스트를 읽기 전에 파악 가능하다
- 대비비(contrast ratio)는 WCAG AA 기준(4.5:1) 이상을 준수한다

### 1.3 다크 모드 기본 (Dark-First)

개발자는 어두운 환경에서 장시간 작업한다. 다크 모드가 기본이며, 라이트 모드는 선택적으로 지원한다.

- 기본 배경은 `#0A0A0F` (near-black)로 눈의 피로를 최소화한다
- 라이트 모드는 동등한 수준으로 설계하되, 다크 모드가 기준 설계다
- 테마 전환은 `prefers-color-scheme` 감지 + 사용자 수동 전환을 모두 지원한다

### 1.4 상태가 항상 보인다

AI 에이전트의 상태(작업 중/대기/완료/오류)는 어느 화면에서든 즉시 식별 가능해야 한다.

- Set 컬러 도트로 상태를 한눈에 파악한다
- 애니메이션(pulse, spin)으로 현재 진행 중인 작업을 명확히 표시한다
- 오류 상태는 즉시 눈에 띄는 빨간 계열로 표시한다

## 2. 컬러 시스템

### 2.1 다크 모드 팔레트

배경부터 텍스트까지 6단계로 구성한다. 각 단계는 역할이 명확히 구분된다.

```
단계         토큰 이름              값              용도
─────────────────────────────────────────────────────────
L0 (최하층)  --color-bg-base        #0A0A0F        앱 전체 배경
L1           --color-bg-elevated    #111118        사이드바, 패널 배경
L2           --color-surface        #1A1A24        카드, 메시지 버블, 입력창
L3           --color-surface-hover  #22222E        호버 상태, 강조 영역
─────────────────────────────────────────────────────────
B1           --color-border         #2A2A38        기본 구분선
B2           --color-border-strong  #3A3A4E        강조 구분선, 포커스 링
─────────────────────────────────────────────────────────
T1           --color-text-primary   #F0F0F8        제목, 강조 텍스트
T2           --color-text-secondary #A0A0B8        본문, 메시지 내용
T3           --color-text-muted     #6B6B85        메타정보, 타임스탬프
T4           --color-text-disabled  #3D3D52        비활성 상태
```

### 2.3 Set 컬러 8색

각 Agent Set(팀)에 고유 색상을 할당한다. 최대 8개 Set을 동시 운영할 수 있다.

| 인덱스 | 이름     | Hex       | 다크 배경용 (muted)  | 용도 예시      |
|--------|----------|-----------|---------------------|----------------|
| 1      | 보라     | `#8B5CF6` | `#2D1B69`           | 아키텍처팀     |
| 2      | 초록     | `#22C55E` | `#0D3320`           | 백엔드팀       |
| 3      | 파랑     | `#3B82F6` | `#0D2451`           | 프론트엔드팀   |
| 4      | 노랑     | `#EAB308` | `#3D2F00`           | QA팀           |
| 5      | 주황     | `#F97316` | `#3D1A00`           | DevOps팀       |
| 6      | 빨강     | `#EF4444` | `#3D0D0D`           | 보안팀         |
| 7      | 핑크     | `#EC4899` | `#3D0D27`           | 디자인팀       |
| 8      | 시안     | `#06B6D4` | `#003040`           | 데이터팀       |

**Set 컬러 사용 규칙:**

- **발신자 이름**: Set 컬러를 전체 채도로 적용 (`color: #8B5CF6`)
- **아바타/도트**: Set 컬러 배경 + 흰색 아이콘
- **메시지 버블 배경**: Set 컬러의 muted 버전 (투명도 15~20%)
- **왼쪽 보더 강조선**: Set 컬러 전체 채도 2px solid
- **상태 인디케이터**: Set 컬러 도트(8px), 작업 중이면 pulse 애니메이션
- **컬러 할당 순서**: Set 생성 순서대로 인덱스 1→8 순환 할당

```typescript
// Set 컬러 팔레트 정의
export const SET_COLORS = [
  { name: '보라',   primary: '#8B5CF6', muted: '#2D1B69', label: 'violet' },
  { name: '초록',   primary: '#22C55E', muted: '#0D3320', label: 'green' },
  { name: '파랑',   primary: '#3B82F6', muted: '#0D2451', label: 'blue' },
  { name: '노랑',   primary: '#EAB308', muted: '#3D2F00', label: 'yellow' },
  { name: '주황',   primary: '#F97316', muted: '#3D1A00', label: 'orange' },
  { name: '빨강',   primary: '#EF4444', muted: '#3D0D0D', label: 'red' },
  { name: '핑크',   primary: '#EC4899', muted: '#3D0D27', label: 'pink' },
  { name: '시안',   primary: '#06B6D4', muted: '#003040', label: 'cyan' },
] as const
```

## 3. 타이포그래피

### 3.1 폰트 패밀리

두 가지 폰트 패밀리를 명확히 분리하여 사용한다.

**일반 텍스트 — system-ui**

```css
--font-sans: ui-sans-serif, system-ui, -apple-system, BlinkMacSystemFont,
             "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif;
```

- 채팅 메시지, UI 레이블, 버튼 등 모든 일반 텍스트에 적용
- 각 OS의 최적화된 시스템 폰트를 그대로 활용하여 렌더링 품질 최대화
- macOS: SF Pro, Windows: Segoe UI, Android: Roboto

**코드/로그 — JetBrains Mono**

```css
--font-mono: "JetBrains Mono", "Fira Code", "Cascadia Code",
             ui-monospace, "Courier New", monospace;
```

- Set 내부 로그, 코드 블록, 파일명, 커밋 해시, 브랜치명에 적용
- JetBrains Mono 우선 로드 (Google Fonts), 미로드 시 Fira Code → 시스템 모노
- 리거처(ligatures) 활성화: `font-feature-settings: "liga" 1, "calt" 1`

```html
<!-- index.html에 폰트 프리로드 추가 -->
<link rel="preconnect" href="https://fonts.googleapis.com" />
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />
<link
  href="https://fonts.googleapis.com/css2?family=JetBrains+Mono:wght@400;500;700&display=swap"
  rel="stylesheet"
/>
```

### 3.2 사이즈 스케일

4px 기반 배수 체계를 사용한다.

| 토큰  | 값    | line-height | 용도                                     |
|-------|-------|-------------|------------------------------------------|
| `xs`  | 11px  | 1.4         | 뱃지, 상태 레이블, 툴팁 내부             |
| `sm`  | 12px  | 1.5         | 메타정보, 타임스탬프, Set 로그 메타      |
| `md`  | 13px  | 1.6         | Set 내부 로그, 코드 블록, 파일명         |
| `base`| 14px  | 1.65        | 채팅 메시지 본문 (기본값)                |
| `lg`  | 15px  | 1.6         | 발신자 이름, 섹션 레이블                 |
| `xl`  | 16px  | 1.5         | 패널 제목, 탭 레이블                     |
| `2xl` | 18px  | 1.4         | 페이지 제목, 모달 제목                   |

### 3.4 줄간격 (line-height)

```css
/* 채팅 메시지: 편안한 읽기 */
.message-body { line-height: 1.65; }

/* 로그/코드: 밀도 높은 정보 */
.log-line     { line-height: 1.5; }
.code-block   { line-height: 1.6; }

/* UI 레이블: 단일 행 */
.ui-label     { line-height: 1.4; }

/* 제목 */
.heading      { line-height: 1.3; }
```

### 3.5 자간 (letter-spacing)

```css
/* 일반 텍스트: 기본값 */
.body-text    { letter-spacing: 0; }

/* 대문자 레이블 (섹션 헤더 등) */
.label-upper  { letter-spacing: 0.06em; text-transform: uppercase; }

/* 모노스페이스: 약간 넓게 */
.mono-text    { letter-spacing: 0.01em; }

/* 뱃지, 상태 태그 */
.badge-text   { letter-spacing: 0.04em; }
```

## 4. 간격 시스템

4px를 기본 단위로 하는 배수 체계다. Tailwind의 기본 간격 스케일(0.25rem = 4px)과 일치한다.

```
토큰     px      rem      용도
────────────────────────────────────────────────────────────
space-0   0px    0        없음
space-1   4px    0.25rem  아이콘↔텍스트 미세 간격
space-2   8px    0.5rem   인라인 요소 간격, 뱃지 패딩
space-3   12px   0.75rem  컴팩트 컴포넌트 패딩 (버튼 sm)
space-4   16px   1rem     기본 패딩 (버튼 md, 카드 내부)
space-5   20px   1.25rem  섹션 내부 요소 간격
space-6   24px   1.5rem   카드 패딩, 패널 패딩
space-8   32px   2rem     섹션 간 간격
space-10  40px   2.5rem   대형 컴포넌트 여백
space-12  48px   3rem     페이지 레벨 여백
space-16  64px   4rem     섹션 구분 여백
```

**컴포넌트별 권장 간격:**

```
채팅 메시지 버블
  padding: 12px 16px (space-3 x space-4)
  메시지 간격: 4px (space-1) — 같은 발신자 연속
  발신자 그룹 간격: 16px (space-4)

패널 (Left / Right)
  padding: 16px (space-4)
  아이템 간격: 8px (space-2)

카드 (태스크, PR)
  padding: 16px (space-4)
  카드 간격: 8px (space-2)

입력창
  padding: 12px 16px (space-3 x space-4)

Bottom Bar
  padding: 0 16px (space-4)
  height: 40px
```

## 5. 그림자 (Elevation)

다크 배경에서는 색상으로 계층을 표현하는 것이 그림자보다 효과적이다. 그림자는 팝오버, 드롭다운, 모달 등 부유하는 UI에만 사용한다.

```
레벨      토큰                    값                                      용도
──────────────────────────────────────────────────────────────────────────────
0 (없음)  shadow-none             none                                    일반 컨테이너
1 (낮음)  shadow-sm               0 1px 2px rgba(0,0,0,0.4)              툴팁, 작은 팝오버
2 (중간)  shadow-md               0 4px 12px rgba(0,0,0,0.5),            드롭다운, 컨텍스트 메뉴
                                  0 1px 3px rgba(0,0,0,0.3)
3 (높음)  shadow-lg               0 8px 24px rgba(0,0,0,0.6),            모달, 다이얼로그
                                  0 2px 8px rgba(0,0,0,0.4)
```

## 6. 모서리 반경

채팅 인터페이스와 개발 도구의 특성을 반영하여 날카롭지도 지나치게 둥글지도 않은 중간 수준을 기본으로 한다.

```
토큰          px      용도
─────────────────────────────────────────────────────
rounded-none  0px     구분선, 인라인 요소
rounded-sm    4px     뱃지, 태그, 작은 버튼
rounded       6px     기본 버튼, 입력창, 카드 (기본값)
rounded-md    8px     패널, 드롭다운, 컨텍스트 메뉴
rounded-lg    12px    모달, 대형 카드
rounded-xl    16px    바텀 시트 (모바일)
rounded-full  9999px  아바타, 상태 도트, 필 타입 뱃지
```

**컴포넌트별 권장 반경:**

```
메시지 버블:   rounded (6px) — 연속 메시지 시 같은 방향 모서리 축소
아바타:        rounded-full (원형)
상태 도트:     rounded-full (원형)
버튼 (기본):   rounded (6px)
버튼 (작은):   rounded-sm (4px)
입력창:        rounded (6px)
카드:          rounded-md (8px)
모달:          rounded-lg (12px)
드롭다운:      rounded-md (8px)
뱃지/태그:     rounded-sm (4px)
```

## 8. 애니메이션

### 8.1 transition 기본값

모든 인터랙티브 요소에 일관된 전환 효과를 적용한다.

```css
/* 기본 전환 — 색상, 배경, 불투명도 */
.transition-base {
  transition-property: color, background-color, border-color, opacity, box-shadow;
  transition-duration: 150ms;
  transition-timing-function: cubic-bezier(0.4, 0, 0.2, 1); /* ease-in-out */
}

/* 변형 포함 전환 (드롭다운 열림/닫힘 등) */
.transition-transform {
  transition-property: color, background-color, border-color, opacity,
                        box-shadow, transform;
  transition-duration: 200ms;
  transition-timing-function: cubic-bezier(0.4, 0, 0.2, 1);
}

/* 레이아웃 전환 (패널 확장/축소) */
.transition-layout {
  transition-property: width, height, max-height, padding;
  transition-duration: 250ms;
  transition-timing-function: cubic-bezier(0.4, 0, 0.6, 1); /* ease-in-out */
}
```

**원칙**: `prefers-reduced-motion: reduce` 를 감지하여 모든 애니메이션을 즉시 전환으로 교체한다.

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

### 11.2 Button 컴포넌트 커스터마이징

`src/components/ui/button.tsx` — size 변형에 개발 도구에 맞는 컴팩트 스케일 추가.

```tsx
import { cva, type VariantProps } from 'class-variance-authority'
import { cn } from '@/lib/utils'
import { Slot } from '@radix-ui/react-slot'
import { forwardRef } from 'react'

const buttonVariants = cva(
  'inline-flex items-center justify-center gap-2 whitespace-nowrap font-medium ' +
  'transition-colors duration-150 focus-visible:outline-none focus-visible:ring-2 ' +
  'focus-visible:ring-ring disabled:pointer-events-none disabled:opacity-40 ' +
  'select-none',
  {
    variants: {
      variant: {
        default:     'bg-primary text-primary-foreground hover:bg-primary/90',
        secondary:   'bg-surface text-text-secondary hover:bg-surface-hover border border-border',
        ghost:       'text-text-secondary hover:bg-surface-hover hover:text-text-primary',
        destructive: 'bg-error text-white hover:bg-error/90',
        link:        'text-info underline-offset-4 hover:underline',
        // Set 컬러 버튼 (동적으로 style 속성으로 색상 주입)
        set:         'text-white hover:opacity-90',
      },
      size: {
        xs: 'h-6 px-2 text-xs rounded-sm',        // 24px — 인라인 액션
        sm: 'h-7 px-3 text-sm rounded-sm',         // 28px — 보조 액션
        md: 'h-8 px-4 text-base rounded',           // 32px — 기본 (기본값)
        lg: 'h-9 px-5 text-lg rounded',             // 36px — 강조 액션
        icon: 'h-8 w-8 rounded',                    // 아이콘 전용
        'icon-sm': 'h-7 w-7 rounded-sm',           // 작은 아이콘
      },
    },
    defaultVariants: {
      variant: 'default',
      size: 'md',
    },
  }
)

export interface ButtonProps
  extends React.ButtonHTMLAttributes<HTMLButtonElement>,
    VariantProps<typeof buttonVariants> {
  asChild?: boolean
}

const Button = forwardRef<HTMLButtonElement, ButtonProps>(
  ({ className, variant, size, asChild = false, ...props }, ref) => {
    const Comp = asChild ? Slot : 'button'
    return (
      <Comp
        className={cn(buttonVariants({ variant, size, className }))}
        ref={ref}
        {...props}
      />
    )
  }
)
Button.displayName = 'Button'

export { Button, buttonVariants }
```

### 11.5 테마 전환 구현

```tsx
// src/hooks/useTheme.ts
import { useEffect, useState } from 'react'

type Theme = 'dark' | 'light' | 'system'

export function useTheme() {
  const [theme, setTheme] = useState<Theme>(() => {
    return (localStorage.getItem('theme') as Theme) ?? 'dark'
  })

  useEffect(() => {
    const root = document.documentElement
    const prefersDark = window.matchMedia('(prefers-color-scheme: dark)').matches

    root.classList.remove('dark', 'light')

    if (theme === 'system') {
      root.classList.add(prefersDark ? 'dark' : 'light')
    } else {
      root.classList.add(theme)
    }

    localStorage.setItem('theme', theme)
  }, [theme])

  return { theme, setTheme }
}
```
