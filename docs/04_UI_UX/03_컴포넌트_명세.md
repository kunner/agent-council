# 컴포넌트 명세서

## 개요

Agent Council 웹 UI (`packages/web`)의 React 컴포넌트 계층 구조, Props 인터페이스, 상태 관리, 이벤트 콜백을 정의한다.
기술 스택: React + TypeScript, Vite, shadcn/ui + Tailwind CSS, Zustand, Firestore onSnapshot, react-resizable-panels.

### 3.1 ChatMessage

채팅방 내 개별 메시지를 렌더링한다. `senderType`에 따라 Human / Leader / System 세 가지 스타일로 분기한다.

```typescript
// packages/web/src/components/chat/ChatMessage.tsx

interface ChatMessageProps {
  /** Firestore에서 조회한 메시지 데이터 */
  message: ChatMessageData
  /** 인라인 액션 버튼 클릭 핸들러 */
  onAction?: (action: MessageAction) => void
  /** 이 메시지에 첨부된 Set 내부 로그 (leader 타입일 때만 표시) */
  setLogs?: LogEntry[]
  /** 시간 표시 형식 (기본: 'HH:mm') */
  timeFormat?: 'HH:mm' | 'full'
}

type MessageAction =
  | { type: 'view_pr';      prUrl: string }
  | { type: 'request_review'; prId: string }
  | { type: 'create_task';  fromMessageId: string; suggestedTitle?: string }
  | { type: 'reply';        replyToId: string }

/** 컴포넌트 내부 상태 */
interface ChatMessageState {
  /** Set 내부 로그 접기/펼치기 */
  isLogsExpanded: boolean
}
```

**타입별 렌더링:**

| `senderType` | 배경 | 아바타 | 이름 컬러 | 특이사항 |
|---|---|---|---|---|
| `human` | `bg-primary/10` (파랑 틴트) | 사용자 프로필 이미지 | `text-primary` | 우측 정렬 (본인 메시지) |
| `leader` | `bg-card` | Set 컬러 원형 이니셜 | Set 고유 색상 | artifacts 링크, SetLogAccordion 포함 |
| `system` | `bg-muted/50` 구분선 스타일 | ⚙️ 아이콘 | `text-muted-foreground` | InlineAction 버튼 포함 |

**메타데이터 표시 조건:**
- `metadata.artifacts` 존재 시: `📎 artifacts: [파일명 목록]` 링크 행 표시
- `metadata.commitHash` 존재 시: `🔗 Branch: [브랜치명] (+N commits)` 표시
- `metadata.pullRequestUrl` 존재 시: `InlineAction` 버튼 렌더링 (PR 보기, 코드 리뷰 요청)
- `metadata.tokenUsage` 존재 시: 메시지 하단에 `text-xs text-muted-foreground`로 토큰 수 표시

```tsx
// 렌더링 조건 예시
function ChatMessage({ message, onAction, setLogs }: ChatMessageProps) {
  const [isLogsExpanded, setIsLogsExpanded] = useState(false)
  const isHuman = message.senderType === 'human'
  const isSystem = message.senderType === 'system'
  const isLeader = message.senderType === 'leader'

  return (
    <div className={cn('flex gap-3 px-4 py-2', isHuman && 'flex-row-reverse')}>
      <MessageAvatar message={message} />
      <div className={cn('flex flex-col gap-1 max-w-[75%]', isHuman && 'items-end')}>
        <MessageHeader message={message} />
        <MessageContent message={message} />
        {message.metadata && <MessageMetaBar metadata={message.metadata} onAction={onAction} />}
        {isLeader && setLogs && setLogs.length > 0 && (
          <SetLogAccordion
            logs={setLogs}
            setName={message.senderName}
            setColor={message.senderColor}
            isExpanded={isLogsExpanded}
            onToggle={() => setIsLogsExpanded(prev => !prev)}
          />
        )}
        {isSystem && message.metadata?.pullRequestUrl && (
          <InlineAction
            pullRequestUrl={message.metadata.pullRequestUrl}
            pullRequestId={message.metadata.pullRequestId}
            fromMessageId={message.id}
            onAction={onAction}
          />
        )}
      </div>
    </div>
  )
}
```

### 3.2 ChatInput

텍스트 입력 및 전송 UI. Enter 단독 전송, Shift+Enter 줄바꿈을 지원한다.

```typescript
// packages/web/src/components/chat/ChatInput.tsx

interface ChatInputProps {
  /** 전송 비활성화 (Council이 응답 중이거나 세션 끊김) */
  disabled?: boolean
  /** 비활성화 이유 메시지 */
  disabledReason?: string
  /** 메시지 전송 콜백 */
  onSend: (content: string) => Promise<void>
  /** 입력 중 타이핑 이벤트 (WebSocket으로 broadcast) */
  onTyping?: () => void
  /** 답장 대상 메시지 (null이면 일반 메시지) */
  replyTo?: ChatMessageData | null
  /** 답장 취소 */
  onCancelReply?: () => void
  /** 플레이스홀더 */
  placeholder?: string
}

/** 컴포넌트 내부 상태 */
interface ChatInputState {
  value: string
  isSending: boolean
}
```

**키보드 단축키:**

| 키 | 동작 |
|---|---|
| `Enter` | 메시지 전송 (빈 문자열이면 무시) |
| `Shift + Enter` | 줄바꿈 삽입 |
| `Escape` | 답장 취소 (`replyTo`가 있을 때) |
| `Ctrl/Cmd + Enter` | 강제 전송 (멀티라인에서도 전송) |

```tsx
function ChatInput({ disabled, onSend, onTyping, replyTo, onCancelReply }: ChatInputProps) {
  const [value, setValue] = useState('')
  const [isSending, setIsSending] = useState(false)
  const textareaRef = useRef<HTMLTextAreaElement>(null)

  const handleKeyDown = (e: KeyboardEvent<HTMLTextAreaElement>) => {
    if (e.key === 'Enter' && !e.shiftKey) {
      e.preventDefault()
      handleSend()
    }
    if (e.key === 'Escape' && replyTo) {
      onCancelReply?.()
    }
    if ((e.metaKey || e.ctrlKey) && e.key === 'Enter') {
      e.preventDefault()
      handleSend()
    }
  }

  const handleSend = async () => {
    const trimmed = value.trim()
    if (!trimmed || isSending || disabled) return
    setIsSending(true)
    try {
      await onSend(trimmed)
      setValue('')
    } finally {
      setIsSending(false)
      textareaRef.current?.focus()
    }
  }

  // 자동 높이 조절 (최소 1행, 최대 6행)
  // ...
}
```

**렌더링 조건:**
- `replyTo` 존재 시: 입력창 위에 답장 미리보기 바 표시
- `disabled` 시: textarea와 전송 버튼 모두 비활성화, `disabledReason` 툴팁 표시
- `isSending` 시: 전송 버튼을 스피너로 교체

### 3.3 ChatRoom

메시지 목록을 표시하는 메인 채팅 컨테이너. Firestore `onSnapshot` 리스너로 실시간 업데이트를 받고, 자동 스크롤과 대화량 증가에 따른 가상화(virtualization)를 처리한다.

```typescript
// packages/web/src/components/chat/ChatRoom.tsx

interface ChatRoomProps {
  projectId: string
  roomId: string
  sets: AgentSet[]
  /** 메시지 인라인 액션 핸들러 */
  onMessageAction: (action: MessageAction) => void
}

/** 컴포넌트 내부 상태 */
interface ChatRoomState {
  messages: ChatMessageData[]
  /** Set별 최신 로그 맵 (setId → LogEntry[]) */
  setLogsMap: Record<string, LogEntry[]>
  isLoading: boolean
  /** 사용자가 스크롤을 위로 올린 상태 (자동 스크롤 비활성) */
  isScrolledUp: boolean
  /** 읽지 않은 새 메시지 수 (스크롤 위로 올렸을 때 표시) */
  unreadCount: number
}
```

**Firestore 리스너 설정:**

```typescript
// 메시지 실시간 구독
useEffect(() => {
  const messagesRef = collection(db, `projects/${projectId}/rooms/${roomId}/messages`)
  const q = query(messagesRef, orderBy('timestamp', 'asc'), limit(200))

  const unsubscribe = onSnapshot(q, (snapshot) => {
    snapshot.docChanges().forEach((change) => {
      if (change.type === 'added') {
        const msg = { id: change.doc.id, ...change.doc.data() } as ChatMessageData
        setMessages(prev => [...prev, msg])
        if (!isScrolledUp) {
          scrollToBottom()
        } else {
          setUnreadCount(prev => prev + 1)
        }
      }
    })
  })
  return unsubscribe
}, [projectId, roomId])
```

**자동 스크롤 동작:**
- 새 메시지 도착 시: `isScrolledUp === false`이면 즉시 `scrollToBottom()` 호출
- `isScrolledUp === true` (사용자가 위로 스크롤한 상태): 하단에 "N개 새 메시지 ↓" 버튼 표시
- 버튼 클릭 또는 화면 최하단 도달 시: `isScrolledUp = false`, `unreadCount = 0`

**가상화 전략:**
- 메시지 수 < 100: 가상화 없이 DOM에 직접 렌더링
- 메시지 수 >= 100: `@tanstack/react-virtual` (또는 유사 라이브러리) 적용하여 뷰포트 기준 윈도우 렌더링
- 메시지 높이가 가변적이므로 `estimateSize` 기본값 80px, `measureElement`로 실측 적용

**Set 로그 맵 구성:**
```typescript
// Set 내부 로그 구독 (활성 Set만)
sets.forEach(set => {
  const logsRef = collection(db, `projects/${projectId}/sets/${set.id}/logs`)
  const q = query(logsRef, orderBy('timestamp', 'desc'), limit(20))
  onSnapshot(q, (snapshot) => {
    const logs = snapshot.docs.map(d => ({ id: d.id, ...d.data() } as LogEntry))
    setSetLogsMap(prev => ({ ...prev, [set.id]: logs }))
  })
})
```

**렌더링 조건:**
- `isLoading` 시: `Skeleton` 컴포넌트 8개 표시
- 메시지 없을 때: 빈 상태 일러스트 + "첫 메시지를 보내 Council을 시작하세요" 안내

### 3.5 TaskBoard

칸반 보드 컴포넌트. 4열(`Backlog`, `In Progress`, `Review`, `Done`) 구조로 태스크를 표시하며, `@hello-pangea/dnd`로 드래그앤드롭을 지원한다.

```typescript
// packages/web/src/components/board/TaskBoard.tsx

interface TaskBoardProps {
  projectId: string
  tasks: Task[]
  sets: AgentSet[]
  /** 태스크 상태 변경 (드래그 또는 버튼) */
  onTaskMove: (taskId: string, newStatus: TaskStatus) => Promise<void>
  /** 태스크 상세 모달 열기 */
  onTaskClick: (task: Task) => void
  /** 새 태스크 생성 모달 열기 */
  onCreateTask: (defaultStatus: TaskStatus) => void
  /** Set 필터 (undefined = 전체) */
  filterSetId?: string
  /** 우선순위 필터 (undefined = 전체) */
  filterPriority?: TaskPriority
}

/** 컴포넌트 내부 상태 */
interface TaskBoardState {
  /** 드래그 중인 태스크 ID */
  draggingTaskId: string | null
}
```

**열 구성:**

```typescript
const BOARD_COLUMNS: { status: TaskStatus; label: string; color: string }[] = [
  { status: 'backlog',      label: 'Backlog',      color: 'border-gray-400' },
  { status: 'in_progress',  label: 'In Progress',  color: 'border-blue-400' },
  { status: 'review',       label: 'Review',       color: 'border-yellow-400' },
  { status: 'done',         label: 'Done',         color: 'border-green-400' },

// 'blocked' 태스크는 원래 열에 그대로 표시하되 빨간 테두리로 강조
```

**드래그앤드롭:**

```typescript
import { DragDropContext, Droppable, Draggable, DropResult } from '@hello-pangea/dnd'

const handleDragEnd = async (result: DropResult) => {
  if (!result.destination) return
  const newStatus = result.destination.droppableId as TaskStatus
  const taskId = result.draggableId
  if (result.source.droppableId === newStatus) return  // 같은 열 내 이동은 무시
  await onTaskMove(taskId, newStatus)
}
```

**필터 동작:**
- `filterSetId` 지정 시: 해당 Set에 할당된 태스크 + 미할당 태스크만 표시
- `filterPriority` 지정 시: 해당 우선순위 태스크만 표시
- 두 필터 동시 적용 가능 (AND 조건)

### 3.7 GitPanel

현재 프로젝트의 Git 상태(브랜치, PR 목록)를 표시한다. PR 상세를 인라인으로 펼쳐 볼 수 있다.


**표시 섹션:**

1. **main 브랜치 요약**: 최신 커밋 수, 마지막 커밋 메시지
2. **열린 PR 목록** (`status: 'open' | 'reviewing' | 'approved'`):
   - PR 번호, 제목, Set 컬러 도트, diff 통계(`+N -M | K files`)
   - `[보기]` 버튼: `expandedPrId` 토글 (인라인 펼침)
   - `[머지]` 버튼: `status === 'approved'`일 때만 활성화
3. **PR 인라인 상세** (`expandedPrId === pr.id`):
   - PR 메타 정보 전체
   - `reviewNotes` 목록
   - `[코드 리뷰 요청]` 버튼
4. **브랜치 목록**: Set별 브랜치명 + 커밋 수 차이 (`ahead N` 배지)

```tsx
function GitPanel({ pullRequests, sets, onMergePR, onViewPR, onRequestReview }: GitPanelProps) {
  const [expandedPrId, setExpandedPrId] = useState<string | null>(null)

  const openPRs = pullRequests.filter(pr =>
    ['open', 'reviewing', 'approved'].includes(pr.status)
  )

  return (
    <div className="flex flex-col gap-4 p-3 overflow-y-auto">
      {/* 열린 PR 섹션 */}
      <section>
        <h4 className="text-xs font-semibold uppercase text-muted-foreground mb-2">
          열린 PR ({openPRs.length})
        </h4>
        {openPRs.map(pr => (
          <PRRow
            key={pr.id}
            pr={pr}
            isExpanded={expandedPrId === pr.id}
            onToggleExpand={() =>
              setExpandedPrId(prev => (prev === pr.id ? null : pr.id))
            }
            onMerge={() => onMergePR(pr.id)}
            onRequestReview={() => onRequestReview(pr.id)}
          />
        ))}
        {openPRs.length === 0 && (
          <p className="text-xs text-muted-foreground">열린 PR 없음</p>
        )}
      </section>

      {/* 브랜치 섹션 */}
      <section>
        <h4 className="text-xs font-semibold uppercase text-muted-foreground mb-2">
          브랜치
        </h4>
        {sets.map(set => (
          <BranchRow key={set.id} set={set} />
        ))}
      </section>
    </div>
  )
}
```

```tsx
      <span>{iconMap[log.type]}</span>
      <span className="text-muted-foreground flex-1">{log.content}</span>
```

**버튼 조합 규칙:**

| 조건 | 표시되는 버튼 |
|---|---|
| `pullRequestUrl` 존재 | [PR 보기] |
| `pullRequestId` + `canRequestReview` | [코드 리뷰 요청] |
| `fromMessageId` 존재 | [태스크 등록] |

```tsx
function InlineAction({
  pullRequestUrl, pullRequestId, prStatus, fromMessageId,
  suggestedTaskTitle, canRequestReview, onAction, loadingAction,
}: InlineActionProps) {
  const buttons = []

  if (pullRequestUrl) {
    buttons.push(
      <Button
        key="view_pr"
        size="sm"
        variant="outline"
        className="h-7 text-xs"
        onClick={() => onAction?.({ type: 'view_pr', prUrl: pullRequestUrl })}
        disabled={loadingAction === 'view_pr'}
      >
        PR 보기
      </Button>
    )
  }

  if (pullRequestId && canRequestReview) {
    buttons.push(
      <Button
        key="request_review"
        size="sm"
        variant="outline"
        className="h-7 text-xs"
        onClick={() => onAction?.({ type: 'request_review', prId: pullRequestId })}
        disabled={loadingAction === 'request_review'}
      >
        코드 리뷰 요청
      </Button>
    )
  }

  if (fromMessageId) {
    buttons.push(
      <Button
        key="create_task"
        size="sm"
        variant="outline"
        className="h-7 text-xs"
        onClick={() =>
          onAction?.({
            type: 'create_task',
            fromMessageId,
            suggestedTitle: suggestedTaskTitle,
          })
        }
        disabled={loadingAction === 'create_task'}
      >
        태스크 등록
      </Button>
    )
  }

  if (buttons.length === 0) return null

  return <div className="flex gap-2 mt-1 flex-wrap">{buttons}</div>
}
```

```typescript
  addSet: (set: AgentSet) => void

}

  activeProjectId: null,
  activeProject: null,
  sets: [],

  error: null,

    set({ activeProjectId: projectId, activeProject: project })

    })),

  addSet: (agentSet) =>
    set(state => ({ sets: [...state.sets, agentSet] })),

  removeSet: (setId) =>
    set(state => ({ sets: state.sets.filter(s => s.id !== setId) })),

  setTasks: (tasks: Task[]) => void
  updateTask: (taskId: string, patch: Partial<Task>) => void
  addTask: (task: Task) => void
  setPullRequests: (prs: PullRequest[]) => void
  updatePR: (prId: string, patch: Partial<PullRequest>) => void
  setFilterSetId: (setId: string | undefined) => void
  setFilterPriority: (priority: TaskPriority | undefined) => void
}

/** 파생 셀렉터 */
export const selectFilteredTasks = (state: TaskState): Task[] => {
  let tasks = state.tasks
  if (state.filterSetId !== undefined) {
    tasks = tasks.filter(
      t => t.assignedSetId === state.filterSetId || t.assignedSetId === undefined
    )
  }
  if (state.filterPriority !== undefined) {
    tasks = tasks.filter(t => t.priority === state.filterPriority)
  }
  return tasks
}

export const useTaskStore = create<TaskState & TaskActions>((set) => ({
  tasks: [],
  pullRequests: [],
  filterSetId: undefined,
  filterPriority: undefined,
  isLoading: false,

  setTasks: (tasks) => set({ tasks }),

  updateTask: (taskId, patch) =>
    set(state => ({
      tasks: state.tasks.map(t => t.id === taskId ? { ...t, ...patch } : t),
    })),

  addTask: (task) =>
    set(state => ({ tasks: [...state.tasks, task] })),

  setPullRequests: (prs) => set({ pullRequests: prs }),

  updatePR: (prId, patch) =>
    set(state => ({
      pullRequests: state.pullRequests.map(pr => pr.id === prId ? { ...pr, ...patch } : pr),
    })),

  setFilterSetId: (setId) => set({ filterSetId: setId }),

  setFilterPriority: (priority) => set({ filterPriority: priority }),
}))
```
