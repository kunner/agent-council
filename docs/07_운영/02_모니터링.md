# 02. 모니터링

## 개요

Agent Council은 Oracle Cloud Ampere(ARM) 서버에서 Council Server를 운영하고, Firebase와 Cloudflare를 외부 서비스로 활용한다. 월 비용 $0 원칙에 따라 모니터링도 무료 도구 중심으로 구성한다.

### 모니터링 레이어 구성

```
┌─────────────────────────────────────────────────────────────┐
│  Layer 1 — 가용성 (Uptime Kuma, 셀프호스트)                   │
│  서버/엔드포인트/WebSocket 살아있는지 5분 간격 확인              │
│  이상 감지 시 Discord/Telegram 즉시 알림                       │
├─────────────────────────────────────────────────────────────┤
│  Layer 2 — 메트릭 & 로그 (Grafana Cloud 무료 플랜)             │
│  CPU/메모리/디스크 추세, 애플리케이션 로그 장기 보관             │
│  Promtail(로그) + Node Exporter(메트릭) → Grafana Cloud       │
├─────────────────────────────────────────────────────────────┤
│  Layer 3 — 애플리케이션 로그 (Council Server 내부)             │
│  Claude Code 세션 이벤트, 토큰 사용량, Firestore 호출 추적      │
├─────────────────────────────────────────────────────────────┤
│  Layer 4 — 비용 추적 (Firestore usage 컬렉션)                  │
│  API 토큰 사용량, Firestore 읽기/쓰기 집계, 월별 리포트          │
└─────────────────────────────────────────────────────────────┘
```

### 무료 한도 요약

| 도구 | 무료 한도 | 비고 |
|---|---|---|
| Uptime Kuma | 제한 없음 (셀프호스트) | Oracle Ampere에서 Docker 실행 |
| Grafana Cloud | 로그 50GB/월, 메트릭 10K 시리즈 | 개인/소규모에 충분 |
| Firebase Console | 무료 | Firestore 사용량 내장 대시보드 |

## 1. Uptime Kuma — 가용성 모니터링

### 1.1 Docker로 설치 (Oracle Ampere)

Oracle Ampere는 ARM64 아키텍처다. Uptime Kuma는 공식 이미지가 멀티아치를 지원하므로 별도 설정 없이 동작한다.

```bash
# 데이터 디렉토리 생성
mkdir -p /opt/uptime-kuma/data

# Docker Compose 파일 작성
cat > /opt/uptime-kuma/docker-compose.yml << 'EOF'
services:
  uptime-kuma:
    image: louislam/uptime-kuma:1
    container_name: uptime-kuma
    restart: unless-stopped
    ports:
      - "3002:3001"   # 호스트 3002 → 컨테이너 3001 (Council Server 3001과 충돌 방지)
    volumes:
      - ./data:/app/data
    environment:
      - TZ=Asia/Seoul
EOF

# 실행
cd /opt/uptime-kuma
docker compose up -d

# 상태 확인
docker compose ps
docker compose logs -f --tail=50
```

초기 접속 후 관리자 계정을 생성한다. Nginx 리버스 프록시를 통해 `https://status.yourdomain.com`으로 외부 노출하는 것을 권장한다.

```nginx
# /etc/nginx/conf.d/uptime-kuma.conf
server {
    listen 443 ssl;
    server_name status.yourdomain.com;

    ssl_certificate     /etc/letsencrypt/live/yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/yourdomain.com/privkey.pem;

    location / {
        proxy_pass         http://localhost:3002;
        proxy_http_version 1.1;
        proxy_set_header   Upgrade $http_upgrade;
        proxy_set_header   Connection "upgrade";
        proxy_set_header   Host $host;
        proxy_set_header   X-Real-IP $remote_addr;
        proxy_cache_bypass $http_upgrade;
    }
}
```

## 2. Grafana Cloud — 메트릭 & 로그

### 2.1 계정 설정

1. https://grafana.com/auth/sign-up 에서 무료 계정 생성
2. 스택 생성: 이름 `agent-council`, 리전 `ap-southeast-1` (싱가포르, 서버와 가까운 곳 선택)
3. 다음 정보를 기록한다.
   - Grafana URL: `https://agent-council.grafana.net`
   - Prometheus Remote Write URL (메트릭)
   - Loki URL (로그)
   - API 키 (Grafana Cloud API 키)

홈 → Connections → Add new connection → Grafana Agent에서 ARM용 설치 스크립트를 확인할 수 있다.

### 2.2 Grafana Alloy (Agent) 설치 — ARM64

Grafana Agent는 Alloy로 통합되었다. ARM64 패키지를 설치한다.

```bash
# Grafana APT 저장소 추가
wget -q -O - https://apt.grafana.com/gpg.key | gpg --dearmor | \
  sudo tee /etc/apt/keyrings/grafana.gpg > /dev/null

echo "deb [signed-by=/etc/apt/keyrings/grafana.gpg] https://apt.grafana.com stable main" | \
  sudo tee /etc/apt/sources.list.d/grafana.list

# 설치
sudo apt-get update
sudo apt-get install -y alloy

# 서비스 활성화
sudo systemctl enable alloy
sudo systemctl start alloy
sudo systemctl status alloy
```

### 2.3 Alloy 설정 파일

`/etc/alloy/config.alloy`를 다음과 같이 작성한다.

```hcl
// /etc/alloy/config.alloy

// ── 로그 수집 (Promtail 역할) ──────────────────────────────────
loki.write "grafana_cloud" {
  endpoint {
    url = "https://logs-prod-ap-southeast1.grafana.net/loki/api/v1/push"
    basic_auth {
      username = "<LOKI_USER_ID>"
      password = "<GRAFANA_CLOUD_API_KEY>"
    }
  }
}

// Council Server 로그 수집
// Docker json-file 드라이버를 사용하므로 loki.source.docker로 수집한다.
// (docker-compose.yml에서 logging.driver: "json-file" 설정 필요)
// 별도 파일 기반 수집(/var/log/council-server/)은 사용하지 않는다.
loki.source.docker "council_server_logs" {
  host       = "unix:///var/run/docker.sock"
  targets    = [{
    __meta_docker_container_name = "/council-server",
  }]
  forward_to = [loki.write.grafana_cloud.receiver]
  labels     = { job = "council-server", host = constants.hostname, env = "production" }
}

// Nginx 로그 수집
local.file_match "nginx_logs" {
  path_targets = [{
    __path__ = "/var/log/nginx/*.log",
    job      = "nginx",
    host     = constants.hostname,
  }]
}

loki.source.file "nginx" {
  targets    = local.file_match.nginx_logs.targets
  forward_to = [loki.write.grafana_cloud.receiver]
}

// 기타 Docker 컨테이너 로그 수집 (Uptime Kuma 등)
loki.source.docker "docker_other_logs" {
  host       = "unix:///var/run/docker.sock"
  targets    = []  // council-server 외 나머지 컨테이너 자동 감지
  forward_to = [loki.write.grafana_cloud.receiver]
  labels     = { job = "docker", host = constants.hostname }
}

// ── 메트릭 수집 ───────────────────────────────────────────────
prometheus.remote_write "grafana_cloud" {
  endpoint {
    url = "https://prometheus-prod-ap-southeast1.grafana.net/api/prom/push"
    basic_auth {
      username = "<PROMETHEUS_USER_ID>"
      password = "<GRAFANA_CLOUD_API_KEY>"
    }
  }
}

// 서버 리소스 메트릭 (Node Exporter)
prometheus.exporter.unix "node" {
  // CPU, 메모리, 디스크, 네트워크
}

prometheus.scrape "node_exporter" {
  targets    = prometheus.exporter.unix.node.targets
  forward_to = [prometheus.remote_write.grafana_cloud.receiver]
  scrape_interval = "30s"
  job_name = "node-exporter"
}

// Council Server 메트릭 (커스텀 /metrics 엔드포인트)
prometheus.scrape "council_server" {
  targets = [{
    __address__ = "localhost:3001",
    job         = "council-server",
  }]
  metrics_path   = "/metrics"
  scrape_interval = "30s"
  forward_to     = [prometheus.remote_write.grafana_cloud.receiver]
}
```

설정 적용:
```bash
sudo systemctl restart alloy
sudo journalctl -u alloy -f
```

### 2.4 Council Server Prometheus 메트릭 노출

Council Server에 `prom-client`를 추가하여 `/metrics` 엔드포인트를 노출한다.

```bash
cd packages/server
npm install prom-client
```

```typescript
// packages/server/src/metrics.ts
import client from 'prom-client'

// 기본 메트릭 수집 활성화 (CPU, 메모리, 이벤트 루프 등)
client.collectDefaultMetrics({ prefix: 'council_' })

// ── 커스텀 메트릭 정의 ────────────────────────────────────────

// Claude Code 세션 수
export const activeSessionsGauge = new client.Gauge({
  name: 'council_active_sessions_total',
  help: '현재 활성 Claude Code 세션 수',
  labelNames: ['project_id'],
})

// 세션 상태별 카운터
export const sessionEventsCounter = new client.Counter({
  name: 'council_session_events_total',
  help: 'Claude Code 세션 이벤트 수',
  labelNames: ['event_type'], // started, ended, error, timeout
})

// API 응답 시간 히스토그램
export const httpRequestDuration = new client.Histogram({
  name: 'council_http_request_duration_seconds',
  help: 'HTTP 요청 처리 시간',
  labelNames: ['method', 'route', 'status_code'],
  buckets: [0.01, 0.05, 0.1, 0.3, 0.5, 1, 2, 5],
})

// WebSocket 연결 수
export const wsConnectionsGauge = new client.Gauge({
  name: 'council_ws_connections_total',
  help: '현재 WebSocket 연결 수',
})

// Firestore 작업 카운터
export const firestoreOpsCounter = new client.Counter({
  name: 'council_firestore_operations_total',
  help: 'Firestore 작업 수',
  labelNames: ['operation_type'], // read, write, delete
})

// 토큰 사용량 카운터
export const tokenUsageCounter = new client.Counter({
  name: 'council_token_usage_total',
  help: '총 Claude API 토큰 사용량',
  labelNames: ['model', 'project_id'],
})

// /metrics 엔드포인트 핸들러
export async function metricsHandler(req: any, res: any) {
  res.set('Content-Type', client.register.contentType)
  res.end(await client.register.metrics())
}
```

```typescript
// packages/server/src/index.ts
import { metricsHandler, httpRequestDuration } from './metrics'

// 메트릭 엔드포인트 등록 (내부 네트워크에서만 접근하도록 Nginx에서 차단)
app.get('/metrics', metricsHandler)

// HTTP 요청 시간 미들웨어
app.use((req, res, next) => {
  const end = httpRequestDuration.startTimer({
    method: req.method,
    route: req.path,
  })
  res.on('finish', () => {
    end({ status_code: res.statusCode })
  })
  next()
})
```

Nginx에서 `/metrics` 외부 접근 차단:
```nginx
location /metrics {
    allow 127.0.0.1;
    deny all;
}
```

### 2.5 Grafana Cloud 대시보드 구성

Grafana Cloud에서 새 대시보드를 만들어 다음 패널을 구성한다.

#### 대시보드 1: 서버 리소스

```
패널 1: CPU 사용률 (%)
  쿼리 (Prometheus):
  100 - (avg(rate(node_cpu_seconds_total{mode="idle"}[5m])) * 100)

패널 2: 메모리 사용률 (%)
  쿼리:
  (1 - (node_memory_MemAvailable_bytes / node_memory_MemTotal_bytes)) * 100

패널 3: 디스크 사용률 (%)
  쿼리:
  (1 - (node_filesystem_avail_bytes{mountpoint="/"} / node_filesystem_size_bytes{mountpoint="/"})) * 100

패널 4: 네트워크 트래픽 (bytes/s)
  쿼리 (수신):
  rate(node_network_receive_bytes_total{device="ens3"}[5m])
  쿼리 (송신):
  rate(node_network_transmit_bytes_total{device="ens3"}[5m])
```

#### 대시보드 2: Council Server 애플리케이션

```
패널 1: 활성 Claude Code 세션 수
  쿼리:
  council_active_sessions_total

패널 2: 세션 이벤트 (시작/종료/에러)
  쿼리:
  rate(council_session_events_total[5m])

패널 3: API 응답 시간 (p50, p90, p99)
  쿼리 (p99):
  histogram_quantile(0.99, rate(council_http_request_duration_seconds_bucket[5m]))

패널 4: API 에러율 (5xx)
  쿼리:
  rate(council_http_request_duration_seconds_count{status_code=~"5.."}[5m])
  /
  rate(council_http_request_duration_seconds_count[5m])

패널 5: WebSocket 연결 수
  쿼리:
  council_ws_connections_total

패널 6: 총 토큰 사용량 (일별 누적)
  쿼리:
  increase(council_token_usage_total[24h])
```

#### 대시보드 3: 로그 탐색 (Loki)

Explore 탭에서 다음 LogQL로 로그를 조회한다.

```logql
# Council Server 에러 로그
{job="council-server"} |= "ERROR"

# 특정 프로젝트의 Claude Code 세션 로그
{job="council-server"} | json | project_id="<PROJECT_ID>" | event_type="session"

# Nginx 접근 로그 (5xx 필터링)
{job="nginx"} | pattern `<ip> - - [<_>] "<method> <path> <_>" <status> <_>` | status >= 500

# 토큰 사용량 로그
{job="council-server"} | json | event_type="token_usage" | line_format "{{.project_id}} {{.tokens}}"
```

## 3. 애플리케이션 레벨 로깅

### 3.1 로그 레벨 정의

| 레벨 | 용도 | 예시 |
|---|---|---|
| `debug` | 개발 중 상세 추적, 프로덕션에서 비활성 | Firestore 쿼리 파라미터, WebSocket 프레임 내용 |
| `info` | 정상 비즈니스 이벤트 | 세션 시작, 메시지 전송, PR 생성 |
| `warn` | 잠재적 문제, 즉각 조치 불필요 | 재시도 성공, 느린 응답, 한도 80% 도달 |
| `error` | 처리 실패, 즉각 대응 필요 | 세션 비정상 종료, Firestore 쓰기 실패, 인증 오류 |

### 3.2 구조화된 JSON 로그 포맷

모든 로그는 JSON 형식으로 출력하여 Loki에서 파싱 가능하게 한다.

```typescript
// packages/server/src/logger.ts
import pino from 'pino'

export const logger = pino({
  level: process.env.LOG_LEVEL || 'info',
  formatters: {
    level: (label) => ({ level: label }),
  },
  base: {
    service: 'council-server',
    version: process.env.npm_package_version,
    host: process.env.HOSTNAME,
  },
  timestamp: pino.stdTimeFunctions.isoTime,
  // 프로덕션: JSON, 개발: pretty
  transport: process.env.NODE_ENV !== 'production'
    ? { target: 'pino-pretty', options: { colorize: true } }
    : undefined,
})
```

출력 예시 (JSON):
```json
{
  "level": "info",
  "time": "2026-03-23T14:32:00.000Z",
  "service": "council-server",
  "version": "0.1.0",
  "event_type": "session_started",
  "project_id": "proj_abc123",
```

### 4.1 Firestore 사용량 추적

Firestore 읽기/쓰기/저장 사용량은 Firebase Console(무료)에서 기본 제공된다. 추가로 Council Server에서 자체 집계를 수행한다.

```typescript
// packages/server/src/firebase/usage-tracker.ts
import { db } from './firestore'
import { increment, serverTimestamp } from 'firebase-admin/firestore'
import { firestoreOpsCounter } from '../metrics'

const USAGE_COLLECTION = 'usage'

export async function trackFirestoreRead(userId: string, projectId: string, count = 1) {
  firestoreOpsCounter.inc({ operation_type: 'read' }, count)

  const now = new Date()
  const monthKey = `${now.getFullYear()}-${String(now.getMonth() + 1).padStart(2, '0')}`

  await db
    .collection(USAGE_COLLECTION)
    .doc(userId)
    .collection('monthly')
    .doc(monthKey)
    .set(
      {
        firestoreReads: increment(count),
        updatedAt: serverTimestamp(),
        byProject: { [projectId]: { reads: increment(count) } },
      },
      { merge: true }
    )
}

export async function trackFirestoreWrite(userId: string, projectId: string, count = 1) {
  firestoreOpsCounter.inc({ operation_type: 'write' }, count)

  const now = new Date()
  const monthKey = `${now.getFullYear()}-${String(now.getMonth() + 1).padStart(2, '0')}`

  await db
    .collection(USAGE_COLLECTION)
    .doc(userId)
    .collection('monthly')
    .doc(monthKey)
    .set(
      {
        firestoreWrites: increment(count),
        updatedAt: serverTimestamp(),
        byProject: { [projectId]: { writes: increment(count) } },
      },
      { merge: true }
    )
}
```

```typescript
// packages/server/src/adapters/token-tracker.ts (확장)
import { db } from '../firebase/firestore'
import { increment, serverTimestamp } from 'firebase-admin/firestore'

export async function trackTokenUsageWithPersistence(params: {
```

### 4.3 월별 리포트 자동 생성

매월 1일 자정에 전월 리포트를 생성하는 크론 작업을 Council Server에 추가한다.

```typescript
// packages/server/src/jobs/monthly-report.ts
import cron from 'node-cron'
import { db } from '../firebase/firestore'
import { logger } from '../logger'

// 매월 1일 00:10 KST 실행
cron.schedule('10 15 1 * *', generateMonthlyReport, {
  timezone: 'Asia/Seoul',
})

async function generateMonthlyReport() {
  const now = new Date()
  // 전월 키
  const prevMonth = new Date(now.getFullYear(), now.getMonth() - 1, 1)
  const monthKey = `${prevMonth.getFullYear()}-${String(prevMonth.getMonth() + 1).padStart(2, '0')}`

  logger.info({ event_type: 'monthly_report_start', month: monthKey }, '월별 리포트 생성 시작')

  try {
    const usersSnapshot = await db.collection('usage').get()

    for (const userDoc of usersSnapshot.docs) {
      const monthlyDoc = await userDoc.ref.collection('monthly').doc(monthKey).get()
      if (!monthlyDoc.exists) continue

      const data = monthlyDoc.data()!

      const report = {
        month: monthKey,
        userId: userDoc.id,
        totalTokens: data.totalTokens ?? 0,
        totalSessions: data.totalSessions ?? 0,
        firestoreReads: data.firestoreReads ?? 0,
        firestoreWrites: data.firestoreWrites ?? 0,
        byProject: data.byProject ?? {},
        // Firestore 무료 한도 대비 사용률
        firestoreReadUsagePercent: Math.round(((data.firestoreReads ?? 0) / 1500000) * 100), // 일 50K × 30일
        firestoreWriteUsagePercent: Math.round(((data.firestoreWrites ?? 0) / 600000) * 100), // 일 20K × 30일
        generatedAt: new Date().toISOString(),
      }

      // 리포트를 Firestore에 저장
      await userDoc.ref.collection('monthly').doc(monthKey).set(
        { report },
        { merge: true }
      )

      logger.info({
        event_type: 'monthly_report_generated',
        month: monthKey,
        user_id: userDoc.id,
        total_tokens: report.totalTokens,
        firestore_read_usage_pct: report.firestoreReadUsagePercent,
      }, '월별 리포트 생성 완료')
    }
  } catch (err) {
    logger.error({ err, event_type: 'monthly_report_failed', month: monthKey }, '월별 리포트 생성 실패')
  }
}
```

```typescript
// 매시간 실행하여 당일 누적 사용량 확인
cron.schedule('0 * * * *', checkDailyUsageThreshold)

async function checkDailyUsageThreshold() {
  const today = new Date().toISOString().slice(0, 10) // YYYY-MM-DD
  const dailyReads = await getDailyFirestoreReads(today)
  const dailyWrites = await getDailyFirestoreWrites(today)

  const READ_LIMIT = 50_000
  const WRITE_LIMIT = 20_000
  const WARN_THRESHOLD = 0.8

  if (dailyReads > READ_LIMIT * WARN_THRESHOLD) {
    logger.warn({
      event_type: 'firestore_limit_warning',
      metric: 'reads',
      current: dailyReads,
      limit: READ_LIMIT,
      percent: Math.round((dailyReads / READ_LIMIT) * 100),
    }, `Firestore 일 읽기 한도 ${Math.round((dailyReads / READ_LIMIT) * 100)}% 도달`)

    await notifyFirestoreWarning('읽기', dailyReads, READ_LIMIT)
  }

  if (dailyWrites > WRITE_LIMIT * WARN_THRESHOLD) {
    logger.warn({
      event_type: 'firestore_limit_warning',
      metric: 'writes',
      current: dailyWrites,
      limit: WRITE_LIMIT,
      percent: Math.round((dailyWrites / WRITE_LIMIT) * 100),
    }, `Firestore 일 쓰기 한도 ${Math.round((dailyWrites / WRITE_LIMIT) * 100)}% 도달`)

    await notifyFirestoreWarning('쓰기', dailyWrites, WRITE_LIMIT)
  }
}

async function notifyFirestoreWarning(type: string, current: number, limit: number) {

}
```
