## 1. Oracle Cloud Ampere 서버 초기 셋업

### 1.1 인스턴스 생성

Oracle Cloud Always Free Tier의 Ampere A1 (ARM) 인스턴스를 생성한다.

**OCI 콘솔 → Compute → Instances → Create Instance**

| 항목 | 설정값 |
|---|---|
| Image | Canonical Ubuntu 22.04 |
| Shape | VM.Standard.A1.Flex |
| OCPU | 4 |
| Memory | 24 GB |
| Boot Volume | 200 GB |
| Architecture | ARM (aarch64) |

**네트워크 설정:**
- Public IP: 자동 할당 (Always Free 고정 IP도 가능)
- VCN: 새로 생성 또는 기존 VCN 사용
- Subnet: Public Subnet

**SSH 키:**
- 로컬에서 키 생성: `ssh-keygen -t ed25519 -C "oracle-ampere"`
- 공개키(`~/.ssh/oracle-ampere.pub`) 내용을 콘솔에 붙여넣기

### 1.2 보안 그룹 (Security List / Network Security Group)

OCI 콘솔 → VCN → Security Lists → Default Security List에서 Ingress 규칙 추가:

| 소스 | 프로토콜 | 포트 | 설명 |
|---|---|---|---|
| 0.0.0.0/0 | TCP | 22 | SSH |
| 0.0.0.0/0 | TCP | 80 | HTTP |
| 0.0.0.0/0 | TCP | 443 | HTTPS |

> **중요**: Oracle Cloud는 보안 그룹 외에 인스턴스 내부의 `iptables`도 제어한다. 아래 OS 방화벽 설정을 반드시 수행해야 한다.

### 1.3 SSH 접속 및 초기 설정

```bash
# 로컬에서 접속
ssh -i ~/.ssh/oracle-ampere ubuntu@<PUBLIC_IP>

# OS 방화벽 포트 허용 (Ubuntu 22.04 기본 iptables 룰 제거)
sudo iptables -F
sudo netfilter-persistent save

# 또는 ufw 사용
sudo ufw allow 22/tcp
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable

# 시스템 업데이트
sudo apt update && sudo apt upgrade -y

# 기본 도구 설치
sudo apt install -y curl wget git vim htop unzip build-essential
```

**~/.ssh/config** (로컬 편의 설정):
```
Host oracle-ampere
    HostName <PUBLIC_IP>
    User ubuntu
    IdentityFile ~/.ssh/oracle-ampere
    ServerAliveInterval 60
```

이후 `ssh oracle-ampere` 로 접속 가능.

### 2.2 실제 nginx.conf 예시

`/etc/nginx/sites-available/council` 파일을 생성한다.

```nginx
# /etc/nginx/sites-available/council

# HTTP → HTTPS 리다이렉트
server {
    listen 80;
    listen [::]:80;
    server_name council.yourdomain.com;

    location /.well-known/acme-challenge/ {
        root /var/www/certbot;
    }

    location / {
        return 301 https://$host$request_uri;
    }
}

# HTTPS 메인 서버
server {
    listen 443 ssl http2;
    listen [::]:443 ssl http2;
    server_name council.yourdomain.com;

    # SSL 인증서 (Let's Encrypt)
    ssl_certificate /etc/letsencrypt/live/council.yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/council.yourdomain.com/privkey.pem;
    include /etc/letsencrypt/options-ssl-nginx.conf;
    ssl_dhparam /etc/letsencrypt/ssl-dhparams.pem;

    # gzip 압축
    gzip on;
    gzip_vary on;
    gzip_proxied any;
    gzip_comp_level 6;
    gzip_types
        text/plain
        text/css
        text/xml
        text/javascript
        application/json
        application/javascript
        application/xml
        application/xml+rss
        application/vnd.ms-fontobject
        application/x-font-ttf
        font/opentype
        image/svg+xml
        image/x-icon;

    # 보안 헤더
    add_header X-Frame-Options "DENY" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;

    # 로그
    access_log /var/log/nginx/council_access.log;
    error_log  /var/log/nginx/council_error.log;

    # API 리버스 프록시
    location /api/ {
        proxy_pass http://127.0.0.1:3001;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # 타임아웃 (Claude Code 응답이 길 수 있음)
        proxy_connect_timeout 60s;
        proxy_send_timeout    300s;
        proxy_read_timeout    300s;

        # 버퍼 설정
        proxy_buffering off;
        proxy_buffer_size 4k;
    }

    # WebSocket 리버스 프록시
    location /ws {
        proxy_pass http://127.0.0.1:3001;
        proxy_http_version 1.1;

        # WebSocket 업그레이드 헤더
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # WebSocket 타임아웃 (장기 연결 유지, 24시간)
        proxy_connect_timeout  86400s;
        proxy_send_timeout     86400s;
        proxy_read_timeout     86400s;
    }

    # 헬스체크 엔드포인트
    location /health {
        proxy_pass http://127.0.0.1:3001;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        access_log off;
    }

    # 나머지 경로는 모두 API로 (Council Server가 라우팅)
    location / {
        proxy_pass http://127.0.0.1:3001;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

```bash
# sites-enabled에 심볼릭 링크 생성

# 기본 설정 비활성화
sudo rm -f /etc/nginx/sites-enabled/default

# 설정 문법 검사

```

## 3. Docker 구성

### 3.1 Dockerfile (ARM multi-stage build)

`packages/server/Dockerfile`:

```dockerfile
# packages/server/Dockerfile
# ARM64 (aarch64) 최적화 멀티스테이지 빌드

# ── Stage 1: 빌드 ──────────────────────────────────────────
FROM node:20-alpine AS builder

# pnpm 설치
RUN npm install -g pnpm

WORKDIR /app

# 모노레포 루트 패키지 파일 복사
COPY package.json pnpm-workspace.yaml pnpm-lock.yaml ./
COPY packages/shared/package.json ./packages/shared/
COPY packages/server/package.json ./packages/server/

# 의존성 설치 (캐시 최적화: 소스 변경 전에 설치)
RUN pnpm install --frozen-lockfile

# 소스 코드 복사
COPY packages/shared ./packages/shared
COPY packages/server ./packages/server

# shared 빌드
RUN pnpm --filter @agent-council/shared build

# server 빌드
RUN pnpm --filter @agent-council/server build

# ── Stage 2: 런타임 ────────────────────────────────────────
FROM node:20-alpine AS runner

# 런타임 시스템 의존성
RUN apk add --no-cache \
    git \
    openssh-client \
    tmux \
    bash \
    curl

# pnpm 설치 (프로덕션 의존성용)
RUN npm install -g pnpm

# Node.js 프로세스 매니저 (컨테이너 내부 크래시 재시작)
RUN npm install -g pm2

WORKDIR /app

# 모노레포 루트 패키지 파일
COPY package.json pnpm-workspace.yaml ./
COPY packages/shared/package.json ./packages/shared/
COPY packages/server/package.json ./packages/server/

# 프로덕션 의존성만 설치
RUN pnpm install --frozen-lockfile --prod

# 빌드 결과물 복사
COPY --from=builder /app/packages/shared/dist ./packages/shared/dist
COPY --from=builder /app/packages/server/dist ./packages/server/dist

# 비루트 사용자로 실행 (보안)
RUN addgroup -g 1001 -S nodejs && \
    adduser -S council -u 1001 -G nodejs

# workspace 디렉토리 (마운트 포인트)
RUN mkdir -p /workspace /logs && \
    chown -R council:nodejs /workspace /logs /app

USER council

EXPOSE 3001

# 헬스체크
HEALTHCHECK --interval=30s --timeout=10s --start-period=30s --retries=3 \
    CMD curl -f http://localhost:3001/health || exit 1

CMD ["node", "packages/server/dist/index.js"]
```

## 4. Cloudflare 설정

### 4.1 DNS 레코드

Agent Council은 Web UI와 API 서버를 분리된 서브도메인으로 운영한다.

```
council.yourdomain.com      → Cloudflare Pages (Web UI, React SPA)
api.council.yourdomain.com  → Oracle Ampere (Council Server API + WebSocket)
```

Cloudflare 대시보드 → 도메인 선택 → DNS → Records → Add record

| Type | Name | Content | Proxy status | TTL |
|---|---|---|---|---|
| CNAME | council | `<Pages 프로젝트>.pages.dev` | Proxied (주황 구름) | Auto |
| A | api.council | `<Oracle Ampere Public IP>` | Proxied (주황 구름) | Auto |

> **Proxied(주황 구름)** 상태를 선택하면 Cloudflare CDN이 앞에 위치하여 DDoS 방어 및 캐싱이 활성화된다.
> Oracle 서버 IP가 직접 노출되지 않아 보안에 유리하다.
>
> Web UI와 API 서버를 처음부터 다른 서브도메인으로 분리하면 Cloudflare Pages 커스텀 도메인과 Oracle A 레코드가 충돌하지 않는다.

### 4.2 SSL/TLS 모드

Cloudflare 대시보드 → SSL/TLS → Overview

**SSL 모드: Full (strict)** 선택

```
브라우저 ─[HTTPS]→ Cloudflare ─[HTTPS]→ Oracle Nginx ─[HTTP]→ Council Server
```

- `Full (strict)`: Cloudflare ↔ Oracle 구간도 유효한 SSL 인증서 요구 (Let's Encrypt 인증서)
- 이보다 낮은 `Flexible`이나 `Full`은 중간자 공격에 취약하므로 사용하지 않는다.

**추가 보안 설정:**
- SSL/TLS → Edge Certificates → Always Use HTTPS: ON
- SSL/TLS → Edge Certificates → HTTP Strict Transport Security (HSTS): Enable
- SSL/TLS → Edge Certificates → Minimum TLS Version: TLS 1.2

### 4.3 Cloudflare Pages 프로젝트 생성

Cloudflare 대시보드 → Pages → Create a project → Connect to Git

**GitHub 연결 설정:**

| 항목 | 값 |
|---|---|
| Repository | `<GITHUB_USER>/agent-council` |
| Production branch | `main` |
| Build command | `pnpm --filter @agent-council/web build` |
| Build output directory | `packages/web/dist` |
| Root directory | `/` |
| Node.js version | `20` |

**환경 변수 (Pages 설정 → Environment variables):**

```
# Production 환경
VITE_FIREBASE_API_KEY          = AIzaSy...
VITE_FIREBASE_AUTH_DOMAIN      = agent-council-xxxxx.firebaseapp.com
VITE_FIREBASE_PROJECT_ID       = agent-council-xxxxx
VITE_FIREBASE_STORAGE_BUCKET   = agent-council-xxxxx.appspot.com
VITE_FIREBASE_MESSAGING_SENDER_ID = 1234567890
```

### 5.2 firestore.rules 배포

`firebase/firestore.rules` (PLAN.md 6.2 섹션의 규칙):

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // 사용자 문서: 본인만 읽기/쓰기
    match /users/{userId} {
      allow read, write: if request.auth != null
        && request.auth.uid == userId;
    }

    // 프로젝트: 소유자만 접근
    match /projects/{projectId} {
      allow read, write: if request.auth != null
        && resource.data.ownerId == request.auth.uid;

      // 신규 프로젝트 생성
      allow create: if request.auth != null
        && request.resource.data.ownerId == request.auth.uid;

      // Git 설정
      match /git/{docId} {
        allow read, write: if request.auth != null
          && get(/databases/$(database)/documents/projects/$(projectId)).data.ownerId
             == request.auth.uid;
      }

      // Council Room
      match /rooms/{roomId} {
        allow read, write: if request.auth != null
          && get(/databases/$(database)/documents/projects/$(projectId)).data.ownerId
             == request.auth.uid;

        // 메시지: 프로젝트 소유자 읽기/생성
        match /messages/{messageId} {
          allow read, create: if request.auth != null
            && get(/databases/$(database)/documents/projects/$(projectId)).data.ownerId
               == request.auth.uid;
        }
      }

      // Agent Set
      match /sets/{setId} {
        allow read: if request.auth != null
          && get(/databases/$(database)/documents/projects/$(projectId)).data.ownerId
             == request.auth.uid;
        allow write: if request.auth != null
          && get(/databases/$(database)/documents/projects/$(projectId)).data.ownerId
             == request.auth.uid;

        // Set 내부 로그
        match /logs/{logId} {
          allow read: if request.auth != null
            && get(/databases/$(database)/documents/projects/$(projectId)).data.ownerId
               == request.auth.uid;
          allow create: if request.auth != null;
        }
      }

      // 태스크 보드
      match /tasks/{taskId} {
        allow read: if request.auth != null
          && get(/databases/$(database)/documents/projects/$(projectId)).data.ownerId
             == request.auth.uid;
        allow write: if request.auth != null
          && get(/databases/$(database)/documents/projects/$(projectId)).data.ownerId
             == request.auth.uid;
      }

      // PR 추적
      match /pullRequests/{prId} {
        allow read: if request.auth != null
          && get(/databases/$(database)/documents/projects/$(projectId)).data.ownerId
             == request.auth.uid;
        allow write: if request.auth != null
          && get(/databases/$(database)/documents/projects/$(projectId)).data.ownerId
             == request.auth.uid;
      }

      // 스냅샷
      match /snapshots/{snapshotId} {
        allow read: if request.auth != null
          && get(/databases/$(database)/documents/projects/$(projectId)).data.ownerId
             == request.auth.uid;
        allow create: if request.auth != null;
      }
    }

    // 사용량 추적
    match /usage/{userId} {
      allow read, write: if request.auth != null
        && request.auth.uid == userId;

      match /monthly/{month} {
        allow read, write: if request.auth != null
          && request.auth.uid == userId;
      }
    }
  }
}
```

### 6.2 CI/CD 워크플로우 YAML

`.github/workflows/deploy.yml`:

```yaml
name: Deploy

on:
  push:
    branches:
      - main
  pull_request:
    branches:
      - main

jobs:
  # ── 변경 감지 ─────────────────────────────────────────────
  changes:
    runs-on: ubuntu-latest
    outputs:
      web: ${{ steps.filter.outputs.web }}
      server: ${{ steps.filter.outputs.server }}
      firebase: ${{ steps.filter.outputs.firebase }}
    steps:
      - uses: actions/checkout@v4
      - uses: dorny/paths-filter@v3
        id: filter
        with:
          filters: |
            web:
              - 'packages/web/**'
              - 'packages/shared/**'
            server:
              - 'packages/server/**'
              - 'packages/shared/**'
              - 'docker-compose.yml'
              - 'packages/server/Dockerfile'
            firebase:
              - 'firebase/firestore.rules'
              - 'firebase/firestore.indexes.json'

  # ── Web UI 빌드 & Cloudflare Pages 배포 ──────────────────
  deploy-web:
    needs: changes
    if: needs.changes.outputs.web == 'true' || github.event_name == 'push'
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: pnpm/action-setup@v4
        with:
          version: latest

      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'pnpm'

      - name: Install dependencies
        run: pnpm install --frozen-lockfile

      - name: Build shared
        run: pnpm --filter @agent-council/shared build

      - name: Build web
        run: pnpm --filter @agent-council/web build
        env:
          VITE_FIREBASE_API_KEY: ${{ secrets.VITE_FIREBASE_API_KEY }}
          VITE_FIREBASE_AUTH_DOMAIN: ${{ secrets.VITE_FIREBASE_AUTH_DOMAIN }}
          VITE_FIREBASE_PROJECT_ID: ${{ secrets.VITE_FIREBASE_PROJECT_ID }}
          VITE_FIREBASE_STORAGE_BUCKET: ${{ secrets.VITE_FIREBASE_STORAGE_BUCKET }}
          VITE_FIREBASE_MESSAGING_SENDER_ID: ${{ secrets.VITE_FIREBASE_MESSAGING_SENDER_ID }}
          VITE_FIREBASE_APP_ID: ${{ secrets.VITE_FIREBASE_APP_ID }}
          VITE_API_BASE_URL: ${{ secrets.VITE_API_BASE_URL }}
          VITE_WS_URL: ${{ secrets.VITE_WS_URL }}

      - name: Deploy to Cloudflare Pages
        uses: cloudflare/pages-action@v1
        with:
          apiToken: ${{ secrets.CLOUDFLARE_API_TOKEN }}
          accountId: ${{ secrets.CLOUDFLARE_ACCOUNT_ID }}
          projectName: agent-council
          directory: packages/web/dist
          # PR의 경우 프리뷰 URL 자동 생성, main 브랜치는 프로덕션 배포
          gitHubToken: ${{ secrets.GITHUB_TOKEN }}

  # ── Server Docker 이미지 빌드 & Oracle Ampere 배포 ────────
  deploy-server:
    needs: changes
    if: |
      (needs.changes.outputs.server == 'true' || github.event_name == 'push')
      && github.ref == 'refs/heads/main'
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up QEMU (ARM 에뮬레이션)
        uses: docker/setup-qemu-action@v3
        with:
          platforms: linux/arm64

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3

      - name: Log in to GitHub Container Registry
        uses: docker/login-action@v3
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GHCR_TOKEN }}

      - name: Extract metadata
        id: meta
        uses: docker/metadata-action@v5
        with:
          images: ghcr.io/${{ github.repository_owner }}/agent-council-server
          tags: |
            type=ref,event=branch
            type=sha,prefix=sha-
            type=raw,value=latest,enable=${{ github.ref == 'refs/heads/main' }}

      - name: Build and push Docker image (ARM64)
        uses: docker/build-push-action@v5
        with:
          context: .
          file: packages/server/Dockerfile
          platforms: linux/arm64
          push: true
          tags: ${{ steps.meta.outputs.tags }}
          labels: ${{ steps.meta.outputs.labels }}
          cache-from: type=gha
          cache-to: type=gha,mode=max

      - name: Deploy to Oracle Ampere via SSH
        uses: appleboy/ssh-action@v1
        with:
          host: ${{ secrets.ORACLE_HOST }}
          username: ${{ secrets.ORACLE_USER }}
          key: ${{ secrets.ORACLE_SSH_KEY }}
          script: |
            cd /opt/agent-council

            # 최신 이미지 pull
            docker pull ghcr.io/${{ github.repository_owner }}/agent-council-server:latest

            # 무중단 배포: 새 컨테이너 시작 후 구 컨테이너 제거
            docker compose up -d --no-deps server

            # 헬스체크 대기
            echo "Waiting for health check..."
            for i in $(seq 1 12); do
              if curl -sf http://localhost:3001/health; then
                echo "Health check passed"
                break
              fi
              if [ $i -eq 12 ]; then
                echo "Health check failed after 60s"
                docker compose logs server --tail 50
                exit 1
              fi
              sleep 5
            done

            # 미사용 이미지 정리
            docker image prune -f

            echo "Deployment completed successfully"

  # ── Firebase Rules & Indexes 배포 ────────────────────────
  deploy-firebase:
    needs: changes
    if: |
      needs.changes.outputs.firebase == 'true'
      && github.ref == 'refs/heads/main'
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Install Firebase CLI
        run: npm install -g firebase-tools

      - name: Deploy Firestore rules and indexes
        run: firebase deploy --only firestore
        env:
          FIREBASE_TOKEN: ${{ secrets.FIREBASE_TOKEN }}
```


롤백은 30초 이내에 전 세계 Cloudflare 엣지에 반영된다.

### 9.3 Firestore Rules 롤백

```bash
# git 히스토리에서 이전 버전 복원
git log firebase/firestore.rules  # 커밋 히스토리 확인

# 롤백 완료 후 임시 복원 커밋 생성
git add firebase/firestore.rules
git commit -m "revert: rollback firestore rules to <이전_커밋_SHA>"
git push origin main
```

### 9.4 롤백 판단 기준

| 상황 | 대응 |
|---|---|
| 헬스체크 `/health` 응답 없음 | Server 즉시 롤백 |
| WebSocket 연결 지속 실패 | Server 롤백 + Nginx 설정 확인 |
| Web UI 흰 화면/빌드 오류 | Cloudflare Pages UI 롤백 |
| Firestore 권한 오류 | Rules 롤백 |
| 신규 기능 버그 (서비스 영향 없음) | 핫픽스 PR 생성 후 재배포 |
